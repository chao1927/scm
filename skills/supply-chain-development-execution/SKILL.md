# 供应链系统开发执行

## 必读资料

开始任何功能前，按顺序读取：

1. `AGENTS.md`。
2. `docs/09-开发计划/01-开发计划.md` 中对应系统、迭代，以及“接口与类实施包”的任务编号。
3. 对应系统的 `docs/03` 聚合设计、`docs/04` 功能设计、`docs/05` 数据库、`docs/06` 接口、`docs/07` 事件、`docs/08` 实现逻辑。
4. `docs/09-开发计划/02-开发日志.md` 中该系统最近记录。

## 固定执行步骤

1. **定位切片**：确认任务编号、接口族、Controller、应用服务、聚合根、资源库、命令、查询、事件、上下游、状态机和不变量。
2. **先列实现清单**：HTTP/Dubbo 接口、接口层类、应用服务、领域类、资源库、Mapper、Flyway、事件消费者、测试类；不清楚则先更新设计/日志待决项。
3. **按层编码**：接口层不承载业务规则；应用层处理事务、权限、幂等、审计、Outbox；领域层保护不变量；基础设施层实现数据库/MQ/外部客户端。
4. **处理可靠性**：写操作必须幂等；跨系统命令必须超时、重试、熔断和补偿；事件必须 Outbox/Inbox、失败记录、死信/人工重放。
5. **完成验证**：至少运行领域单测、应用服务测试和 API 测试；涉及库存/金额/并发时增加乐观锁和幂等测试；涉及外部系统时增加集成或契约测试。
6. **记录交付**：更新设计差异、接口/事件/DDL；按 `02-开发日志.md` 模板增加日志。未记录日志不得声称完成。

## 强制规则

- 默认中文；类名、方法名、事件名可使用英文业务术语。
- Java 必须多行格式，禁止压缩一行代码；注释说明业务理由、边界和补偿，不写无意义注释。
- 不允许 Controller 直接操作 Mapper，不允许跨系统直接写对方数据库。
- 列表查询必须分页、总数、排序白名单、数据权限；写接口必须权限、数据范围、版本或幂等键。
- 任何删除先定义软删/状态机/历史策略；任何枚举值先同步 DDL、代码和接口文档。
- 每次完成后在最终答复提供：功能编号、关键类/接口、迁移、事件、测试命令、开发日志位置、剩余风险。
