# 架构检查清单

设计架构、数据、集成、权限和可靠性决策时使用本参考。

## 边界决策

确定以下数据的事实源：

| 数据 | 常见归属系统 |
| --- | --- |
| 会计记录 | 财务系统/ERP |
| 商品主数据 | ERP/MDM/PIM |
| 库存执行 | WMS 或 SCM 库存服务 |
| 库存可视化 | SCM 读模型或库存服务 |
| 采购/销售义务 | 取决于范围，可能是 ERP/OMS/SCM |
| 发运跟踪 | TMS/承运商集成 |
| 供应商协同 | SCM/SRM |

## 服务拆分

在不变量清晰前避免过早拆服务。较自然的服务边界通常包括：

- 主数据服务
- 订单编排服务
- 采购服务
- 库存服务
- 仓储集成服务
- 物流服务
- 结算服务
- 合作伙伴协同服务
- 报表/读模型服务
- 集成网关

## 数据设计规则

- 库存变动使用不可变台账行。
- 单据头表和行表分开存储。
- 导入订单、外部回调和库存变动必须使用幂等键。
- 外部 ID 要同时记录系统编码、单据类型和版本。
- 数量要保留单位、基本单位数量和换算快照。
- 金额要保留币种、含税/未税模式、精度和价格来源。
- 受监管流程要包含审计字段和业务操作日志。

## 集成模式

| 场景 | 模式 |
| --- | --- |
| 用户提交命令 | 同步命令、校验、操作日志 |
| 外部状态变化 | 事件或回调，并保证幂等 |
| 大批量导入/导出 | 批处理任务、暂存表、错误报告 |
| 跨系统一致性 | Outbox/Inbox、重试、对账任务 |
| 读多写少的看板 | 物化读模型 |
| 长流程 | Saga/流程管理器，并显式定义补偿动作 |

## 可靠性风险

检查：

- 重复回调或重试风暴
- 事件乱序
- 下游部分失败
- 同一库存上的并发分配
- ERP、WMS、SCM 之间的对账漂移
- 脏主数据污染批量导入
- 跨组织或跨货主权限泄漏
- 单位换算静默错误

## 权限模型

建模授权时考虑：

- 角色/动作权限
- 组织范围
- 仓库/站点范围
- 合作伙伴范围
- 单据归属与分派
- 价格/成本数据的字段级限制
- 按金额、品类、组织、异常类型定义审批权限

## 非功能目标

询问或假设：

- 订单导入延迟
- 库存可视化新鲜度
- 分配吞吐量
- API QPS 和批处理规模
- 审计留存周期
- RPO/RTO
- 报表新鲜度
- 旺季峰值倍数
