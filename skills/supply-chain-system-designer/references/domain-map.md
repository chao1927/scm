# 供应链领域地图

拆解模块、实体、流程、状态或异常路径时使用本参考。

## 能力地图

| 领域 | 能力 |
| --- | --- |
| 主数据 | 商品/SKU、单位换算、包装、仓库/站点、库位、供应商、客户、承运商、组织、税率、币种 |
| 采购 | 请购、寻源、采购订单、供应商确认、ASN、收货、质检、退货、发票匹配 |
| 库存 | 库存台账、可承诺库存、预留、分配、调拨、调整、盘点、批次/序列号、效期、货权 |
| 仓储 | 收货、上架、拣货、包装、发货、补货、波次/批量作业、库位规则、任务派发 |
| 履约 | 销售订单导入、承诺、分配、拆单发运、发货单、取消、欠交、退货 |
| 物流 | 发运、承运商分配、路线、轨迹、运费预估、签收证明、异常处理 |
| 计划 | 需求预测、供应计划、MRP、安全库存、补货点、产能/运力约束 |
| 协同 | 供应商门户、客户门户、确认、单据交换、消息、附件 |
| 结算 | 三单匹配、运费结算、对账、发票、应付/应收交接 |
| 分析 | 库龄、服务水平、OTIF、满足率、供应商绩效、成本、异常 |

## 核心实体

| 实体 | 说明 |
| --- | --- |
| SKU | 区分商品定义、可销售 SKU、可采购 SKU、单位换算、包装层级 |
| 组织 | 公司、事业部、货主、成本中心、法人主体 |
| 合作伙伴 | 供应商、客户、承运商、3PL；支持资质/状态 |
| 订单 | 意图/义务类单据：采购订单、销售订单、调拨订单 |
| 执行单据 | ASN、收货、质检、上架、拣货、包装、发运、交付、退货 |
| 库存余额 | 按 SKU、仓库、库位、批次、货主、状态查询当前库存的读模型 |
| 库存台账 | 只追加的库存变动记录，包含来源单据和变动前后数量 |
| 预留/分配 | 物理移动前，将可用库存承诺给需求 |
| 结算单据 | 对账、发票、应付/应收交接 |

## 库存状态

常见状态：

- 在库
- 可用
- 已预留
- 已分配
- 已拣货
- 在途
- 质检冻结
- 破损
- 过期
- 隔离
- 已退回

除非业务证明三者总是同步变化，否则物理状态、货权和财务状态要分开建模。

## 单据状态模式

为每类单据设计小型状态机：

| 状态 | 含义 |
| --- | --- |
| 草稿 | 可编辑，尚未产生外部义务 |
| 已提交 | 等待审批或确认 |
| 已确认 | 相关方或下游系统已接受 |
| 部分执行 | 已处理部分数量 |
| 已完成 | 所有必要数量/作业已完成 |
| 已取消 | 不允许继续执行 |
| 已关闭 | 财务或运营关闭完成 |

支持部分履约时，行状态要和单头状态分开跟踪。

## 异常检查清单

为以下情况明确行为：

- 部分收货/部分发运
- 超收/短收
- 预留后的订单取消
- SKU 替代
- 批次/序列号不匹配
- 质检拒收
- 库存调整与审批
- 拆分或合并发运
- 承运商延迟
- 外部重复回调
- 对账中的价格/数量差异
