# 05-销售出库业务流程

> 本文只分析销售出库业务，不引入领域驱动设计术语。目标是先把“客户订单如何变成仓库出库任务，库存如何被锁定和扣减，仓库如何拣货、复核、打包、发货，TMS 如何创建运单、跟踪签收和异常”讲清楚。

## 1. 流程目标

销售出库的目标是：客户下单后，系统完成订单审核、库存预占、仓库出库、物流发货，并在发货后扣减库存、回传发货结果、形成费用依据。

```text
客户下单 -> OMS 审单 -> 中央库存预占 -> TMS 创建运单/面单 -> WMS 拣货复核打包发货 -> 中央库存扣减 -> TMS 跟踪签收/异常 -> BMS 物流费用
```

## 2. 参与系统

| 系统 | 参与原因 | 主要处理内容 | 主要数据变化 |
| --- | --- | --- | --- |
| 销售渠道/商城 | 客户下单入口 | 提交订单、支付、接收发货通知 | 渠道订单、支付状态、发货状态 |
| OMS 系统 | 销售履约编排 | 接收订单、审核订单、拆合单、分仓、请求预占、下发出库、回传发货 | 销售订单、履约单、出库指令、订单状态 |
| 主数据系统 | 提供基础资料 | 提供 SKU、客户、地址、仓库、物流商等资料 | 主数据通常只被引用或快照 |
| 中央库存系统 | 控制库存承诺和扣减 | 查询可用库存、预占库存、释放库存、发货扣减 | 库存余额、预占记录、库存流水 |
| TMS/物流系统 | 运输履约 | 匹配物流渠道、创建运单、生成面单、接收发货交接、同步揽收/在途/签收/异常、生成物流费用来源 | 运输任务、运单、面单、物流轨迹、签收记录、物流异常、物流费用来源 |
| WMS 系统 | 仓库出库执行 | 接收出库单、波次、分配库位、拣货、复核、打包、发货交接 | 出库单、拣货任务、包裹、库内库存 |
| BMS/财务系统 | 费用依据 | 根据 WMS 出库作业事实和 TMS 物流费用来源生成费用 | 出库作业费、物流费、费用明细、对账状态 |
| 权限系统 | 控制人员操作范围 | 校验客服、运营、仓库人员权限，记录操作日志 | 操作日志、审计记录 |

```mermaid
flowchart LR

  B --> C["OMS 审核订单：地址/SKU/金额/风控"]
  C --> D{"审核通过？"}
  D -- "否" --> D1["订单进入异常，客服处理"]
  D -- "是" --> E["OMS 创建履约单并分仓"]
  E --> F["OMS 请求中央库存预占"]
  F --> G{"预占成功？"}
  G -- "否" --> G1["缺货处理：换仓/拆单/等货/取消"]
  G -- "是" --> H["TMS 创建运单和面单"]
  H --> I{"运单成功？"}
  I -- "否" --> I1["物流异常：换渠道/重试/人工处理"]
  I -- "是" --> J["OMS 下发出库单到 WMS"]
  J --> K["WMS 生成波次并分配库位"]
  K --> L{"库位分配成功？"}
  L -- "否" --> L1["分配异常，回传 OMS 并释放或调整预占"]
  L -- "是" --> M["WMS 生成拣货任务"]
  M --> N["拣货员按库位拣货"]
  N --> O{"是否短拣？"}
  O -- "是" --> O1["记录短拣，OMS 决策部分发货/补发/取消"]
  O -- "否" --> P["复核员复核商品和数量"]
  P --> Q{"复核通过？"}
  Q -- "否" --> Q1["复核异常，回退处理"]
  Q -- "是" --> R["打包员打包称重贴面单"]
  R --> S["发货员完成物流交接"]
  S --> T["WMS 回传出库已发货"]
  S --> T1["TMS 接收发货交接并跟踪揽收/在途"]
  T --> U["中央库存扣减库存并记录流水"]
  T --> V["OMS 更新订单已发货并通知渠道"]
  T1 --> T2{"物流是否异常？"}
  T2 -- "异常" --> T3["TMS 记录延误/拒收/破损/丢失"]
  T2 -- "正常签收" --> T4["TMS 回传已签收"]
  T3 --> W["BMS 采集物流费用/索赔依据"]
  T4 --> W
```

## 6. 分步骤数据变化

| 步骤      | 发起角色/系统     | 处理系统     | 被修改的数据           | 数据如何变化                             |
| ------- | ----------- | -------- | ---------------- | ---------------------------------- |
| 客户下单    | 客户/渠道       | 渠道、OMS   | 渠道订单、销售订单        | 生成销售订单，记录客户、地址、商品、数量、金额            |
| 订单审核    | OMS/订单专员    | OMS      | 销售订单             | 待审核变为已审核；异常订单进入异常状态                |
| 创建履约单   | OMS         | OMS      | 履约单              | 按仓库、库存、物流策略生成履约单                   |
| 请求库存预占  | OMS         | 中央库存系统   | 库存预占、库存余额        | 可用库存减少，预占数量增加；失败则不允许下发仓库           |
| 创建运单    | OMS/TMS     | TMS/物流系统 | 运输任务、运单、面单       | 生成运输任务、运单号、面单信息和预计时效               |

| 拣货      | 拣货员         | WMS      | 拣货任务、出库单行        | 记录实拣数量；短拣时记录差异                     |
| 复核      | 复核员         | WMS      | 复核记录、出库单         | 核对商品、数量、批次，通过后进入打包                 |
| 打包      | 打包员         | WMS      | 包裹、重量、体积、面单      | 生成包裹信息，绑定运单或面单                     |
| 发货交接    | 发货员         | WMS、TMS  | 出库单、包裹、物流交接记录、运单 | 出库单变为已发货，包裹交给承运商，TMS 运单进入已发运或已揽收   |
| 发货回传    | WMS         | OMS、渠道   | 销售订单、履约单         | OMS 更新已发货并通知渠道                     |
| 物流轨迹和签收 | 承运商/TMS     | TMS、OMS  | 运单、物流轨迹、销售订单、履约单 | 追加轨迹，签收后 OMS 可将订单推进已签收或已完成         |
| 库存扣减    | 中央库存系统      | 中央库存系统   | 库存余额、库存流水        | 预占转扣减，现货数量减少，生成出库流水                |
| 生成费用    | WMS/TMS/BMS | BMS/财务系统 | 物流费用来源、费用明细      | 根据出库作业、包裹重量体积、承运商、物流产品、签收或异常结果生成费用 |

## 7. 关键数据状态变化

| 数据对象 | 典型状态变化 | 业务含义 |
| --- | --- | --- |
| 销售订单 | 待审核 -> 已审核 -> 履约中 -> 已发货 -> 已完成 | 客户订单从接收到履约完成 |
| 履约单 | 待预占 -> 已预占 -> 已下发 -> 已发货 | OMS 把订单转成可执行的仓库履约任务 |
| 库存预占 | 待预占 -> 已预占 -> 已扣减/已释放 | 先锁住库存，发货后扣减，取消或失败时释放 |
| 运单 | 待创建 -> 已创建 -> 已发运/已揽收 -> 运输中 -> 已签收/拒收/异常 | 物流从下单到配送完成或异常 |
| 出库单 | 待分配 -> 待拣货 -> 待复核 -> 待打包 -> 已发货 | WMS 仓内执行过程 |
| 拣货任务 | 待拣货 -> 拣货中 -> 已完成/短拣 | 拣货员按库位实际拿货 |
| 包裹 | 待打包 -> 已打包 -> 已交接 -> 已发货 | 商品完成打包并交给物流 |
| 库存余额 | 可用数量减少、预占数量增加 -> 发货后库存数量减少 | 库存先承诺，后实扣 |
| 库存流水 | 无 -> 新增预占流水、扣减流水或释放流水 | 记录库存为什么变化 |
| 物流费用来源 | 待采集 -> 已采集 -> 已推送 -> 已计费/差异 | TMS 将承运事实转成 BMS 可计费依据 |
| 费用明细 | 待生成 -> 已生成 -> 待对账 | 出库和物流事实可用于计费 |

## 8. 异常场景

| 异常 | 发生位置 | 影响数据 | 处理方式 |
| --- | --- | --- | --- |
| 订单审核失败 | OMS | 销售订单 | 客服修改、取消或人工审核 |
| 库存预占失败 | 中央库存系统 | 履约单、库存预占 | 换仓、拆单、等货、取消或人工处理 |
| 物流下单失败 | TMS/物流系统 | 运单、履约单 | 更换物流渠道、重试或人工处理 |
| 物流轨迹延迟 | TMS/物流系统 | 运单、销售订单 | TMS 补拉轨迹，OMS 保持已发货/运输中，必要时人工查询 |
| 客户拒收/签收异常 | TMS/OMS | 运单、销售订单、售后单 | OMS 根据拒收原因创建售后、退货入库或重新派送 |
| 运输破损/丢失 | TMS/OMS/BMS | 运单、销售订单、费用明细 | 记录异常，按责任方索赔、补发、退款或费用调整 |
| 出库单重复下发 | OMS/WMS | 出库单 | WMS 按出库单号幂等处理，避免重复作业 |
| 库位分配失败 | WMS | 出库单、分配明细 | 重新分配、库存盘点、回传 OMS 处理 |
| 拣货短拣 | WMS | 拣货任务、出库单、履约单 | OMS 决定部分发货、补发、换仓或取消 |
| 复核异常 | WMS | 复核记录、出库单 | 回退拣货或异常处理，禁止发货 |
| 打包异常 | WMS | 包裹、出库单 | 重新打包、换箱、调整重量体积 |
| 发货回传失败 | WMS、OMS | 出库单、销售订单 | 事件重试或人工补偿查询 |
| 库存扣减失败 | 中央库存系统 | 库存余额、库存流水 | 重试扣减，生成异常记录，人工对账 |
| 费用生成失败 | BMS/财务系统 | 费用明细 | 重试生成或人工补录 |
| 物流费用差异 | TMS/BMS | 物流费用来源、费用明细 | BMS 标记差异，TMS/承运商对账后调整 |

## 9. 业务理解要点

1. 销售订单已审核不代表能发货，必须先完成库存预占。
2. 库存预占不等于库存扣减，只有仓库实际发货后才扣减库存。
3. OMS 负责履约编排，WMS 负责仓内执行，中央库存负责库存数量，TMS 负责物流事实。
4. 拣货、复核、打包、发货都可能产生差异，系统必须记录每一步的实际结果。
5. 一个 OMS 订单可能拆成多个履约单，也可能生成多个出库单和多个包裹。
6. WMS 发货交接不等于客户签收，签收、拒收、破损、丢失等事实以 TMS 为准。
7. 发货后要同时影响订单状态、库存余额、物流轨迹和费用明细。

## 10. 销售出库时序图

```mermaid
sequenceDiagram
  autonumber
  actor Customer as 客户/销售渠道
  participant OMS as OMS系统
  participant INV as 中央库存系统
  participant TMS as TMS/物流系统
  participant WMS as WMS系统
  participant BMS as BMS/财务

  Customer->>OMS: 提交/确认订单
  OMS->>OMS: 审核订单并创建履约单
  OMS->>OMS: 履约单：待预占
  OMS->>INV: 请求库存预占
  INV->>INV: 库存预占：待预占 -> 已预占/失败

  alt 库存预占成功
    INV-->>OMS: 回传库存已预占
    OMS->>TMS: 创建运单/面单
    TMS->>TMS: 运单：已创建
    TMS-->>OMS: 回传运单已创建
    OMS->>WMS: 创建出库单
    WMS->>WMS: 出库单：待分配 -> 待拣货 -> 待复核 -> 待发货
    WMS->>WMS: 发货交接
    WMS-->>OMS: 回传出库已发货
    WMS-->>INV: 通知出库已发货
    WMS-->>TMS: 回传包裹已交接承运商
    TMS->>TMS: 运单：已创建 -> 已揽收/运输中
    INV->>INV: 扣减库存，实物库存减少，预占减少
    INV-->>OMS: 回传库存已扣减
    WMS-->>BMS: 发送出库作业计费依据
    TMS-->>BMS: 发送物流费用来源
  else 库存预占失败
    INV-->>OMS: 回传库存预占失败
    OMS->>OMS: 履约单进入缺货/不可履约
  end

  alt WMS短拣或分配失败
    WMS-->>OMS: 回传短拣/分配失败
    OMS->>INV: 释放剩余预占
    INV->>INV: 库存预占：已预占 -> 部分释放/已释放
  end

  TMS-->>OMS: 回传包裹已签收/拒收/物流异常
  OMS->>OMS: 履约单：已发货 -> 已完成/物流异常
```

### 10.1 销售出库动作链

| 顺序 | 动作 | 来源 | 目标 | 主要数据变化 | 幂等依据 |
| --- | --- | --- | --- | --- | --- |
| 1 | 请求库存预占 | OMS | 中央库存系统 | 库存预占：待预占 -> 已预占/失败 | 履约单号 + 行号 + SKU + 仓库 |
| 2 | 回传库存已预占 | 中央库存系统 | OMS | 履约单：待预占 -> 已预占 | 预占单号 |
| 3 | 创建运单/面单 | OMS | TMS、WMS | 运单：无 -> 已创建；面单已生成 | 履约单号 + 物流产品 + 请求号 |
| 4 | 创建出库单 | OMS | WMS | 出库单：无 -> 待分配 | 出库单号 |
| 5 | 回传出库已发货 | WMS | OMS、中央库存、BMS、TMS | 订单发货、库存扣减、TMS 接收交接 | 发货事件号 |
| 6 | 扣减库存 | WMS/库存系统 | 中央库存系统 | 实物库存减少，预占消耗 | 发货事件号 |
| 7 | 回传签收/异常 | TMS | OMS、BMS | 订单进入已签收/物流异常，生成物流费用来源 | 运单号 + 轨迹节点 |
| 8 | 释放预占 | OMS/WMS | 中央库存系统 | 预占释放或部分释放 | 释放请求号 |

## 11. 业务规则与协同边界

| 检查项 | 设计口径 |
| --- | --- |
| 上游前置 | SKU、客户、收货地址、仓库、货主、物流产品、库存维度和销售规则必须可用 |
| 核心边界 | OMS 负责编排履约；中央库存负责预占/释放/扣减；WMS 负责仓内出库作业；TMS 负责运单和签收；BMS 负责作业费、物流费和对账 |
| 关键事件 | 订单已审核、库存已预占、运单已创建、出库单已创建、出库已发货、库存已扣减、运单已签收、物流异常已发生、物流费用来源已生成 |
| 库存规则 | 预占只减少可用，不减少实物；WMS 已发货后中央库存才把预占转扣减；短拣、取消、部分发货要释放剩余预占 |
| 物流规则 | 运单创建失败时不能下发或必须进入物流异常；WMS 发货交接不等于客户签收；拒收、丢失、破损要触发售后、补发、索赔或退款决策 |
| 费用规则 | WMS 包裹重量体积只是计费输入之一，最终物流费用要结合 TMS 承运商、物流产品、线路、签收和异常责任 |
| 幂等规则 | 预占、出库单创建、发货回传、库存扣减、轨迹回调、费用生成必须用履约单号/出库单号/运单号 + 事件号防重复 |
| 权限审计 | 人工审单、地址修改、取消订单、强制发货、异常关闭、库存释放、费用调整必须记录操作日志 |
