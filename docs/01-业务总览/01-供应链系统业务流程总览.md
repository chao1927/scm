# 01 供应链系统业务流程总览

> 本文是供应链系统的全局领域地图。它不先从菜单、表、接口切入，而是先回答：供应链系统解决什么业务问题、有哪些领域边界、哪些业务主体在流转、哪些动作会改变状态、哪些事实会驱动下游系统继续处理。

## 1. 总体定位

供应链系统的核心作用，是把商品从“计划采购、供应商交付、仓库收发、库存可用、订单履约、物流配送、退货逆向、计费结算”这一整条链路数字化、流程化、可追踪化。

从经营视角看，它管理三类流：

| 类型 | 说明 | 典型承载 |
| --- | --- | --- |
| 信息流 | 商品、供应商、订单、库存、作业、物流、费用等信息如何创建、审批、同步、回传 | 主数据、采购、SRM、OMS、WMS、库存、TMS、BMS |
| 实物流 | 商品如何从供应商进入仓库，再从仓库发给客户，或逆向退回 | 采购入库、仓储作业、销售出库、调拨、退货、退供 |
| 资金流 | 采购应付、销售应收、仓储费、物流费、对账、发票、付款如何闭环 | BMS、财务、采购、SRM、TMS |

最小可运行闭环可以理解为：

```text
主数据建档
  -> 采购下单
  -> WMS 收货上架
  -> 中央库存入账
  -> OMS 接单履约
  -> 中央库存预占/扣减
  -> WMS 拣货发货
  -> 物流配送
  -> 售后/结算/报表
```

## 2. DDD 对齐说明

本文属于“业务总览 / 领域地图”层文档，目标是为后续子系统设计提供统一边界，而不是替代字段模型、接口模型或页面设计。

| DDD 项 | 对齐口径 |
| --- | --- |
| 文档层级 | 全局领域地图、端到端业务闭环、子系统边界总览 |
| 核心域 | 中央库存、订单履约、仓储作业、调拨、逆向退货 |
| 支撑域 | 采购、供应商协同、主数据、物流、BMS、财务 |
| 通用域 | 权限、审计、通知、附件、字典、OpenAPI 网关 |
| 限界上下文 | 主数据、供应商、采购、OMS、中央库存、WMS、TMS、BMS、售后、权限、集成 |
| 核心聚合 | SKU、供应商、采购订单、履约单、库存账户、库存预占、入库单、出库单、调拨单、售后单、费用单 |
| 数据主权 | 每个上下文只维护自己拥有的数据，其他系统通过命令、事件、查询模型或快照引用 |
| 命令 | 下单、审核、预占、释放、收货、验收、上架、拣货、发货、退货、计费等请求动作 |
| 领域事件 | 订单已审核、库存已预占、入库已上架、出库已发货、退货已入库、费用已生成等已经发生的业务事实 |
| 查询模型 | 列表、看板、报表、追踪页、详情页可以跨上下文汇总，但不拥有业务事实 |
| 一致性策略 | 单个聚合内强一致；跨上下文通过事件、幂等、重试、对账、补偿保证最终一致 |

设计原则：

- 先按业务责任划分限界上下文，再决定系统、页面、表和接口。
- 跨系统不能直接修改对方数据，只能发送命令或消费领域事件。
- 库存、订单、仓储作业、费用等关键事实必须能追溯到业务单据和流水。
- 列表、报表、看板是查询模型，不应反向承载核心业务规则。

## 3. 系统边界与默认假设

本文默认设计的是端到端供应链平台，而不是单点进销存工具。

默认支持：

| 能力 | 默认假设 |
| --- | --- |
| 商品 | 多 SPU、多 SKU、多条码、多单位、多包装规格 |
| 组织 | 多公司、多部门、多角色、多数据权限 |
| 合作方 | 多供应商、多客户、多货主、多物流商 |
| 仓网 | 多仓库、多库区、多库位，可支持门店或外部仓 |
| 业务流程 | 采购入库、销售出库、调拨、售后退货、退供应商、盘点、调整 |
| 库存 | 实物库存、可用库存、可销售库存、预占、冻结、在途、库存流水 |
| 集成 | 可对接 ERP、OMS、WMS、TMS、BMS、财务、电商平台、物流商 |
| 结算 | 支持采购应付、仓储费用、物流费用、客户账单、供应商对账 |

暂不在本文深入：

- 字段级设计。
- 数据库 DDL。
- 接口入参和出参。
- 页面交互细节。
- 各聚合完整状态机。

这些内容在后续子系统详细设计文档中展开。

## 4. 领域划分

### 4.1 核心域

核心域决定系统能否支撑供应链高质量运行，应优先深入建模。

| 核心域 | 业务目标 | 关键聚合 | 核心不变量 |
| --- | --- | --- | --- |
| 订单履约 | 把销售订单转成可执行、可追踪、可取消/售后的履约过程 | 销售订单、履约单、发货指令 | 一个履约单只能在合法状态下分仓、下发、取消、关闭 |
| 中央库存 | 保护全局库存可用性，防止超卖和重复占用 | 库存账户、库存预占、库存流水 | 可用库存不能小于已预占数量；库存变化必须有流水 |
| 仓储作业 | 把入库、上架、拣货、复核、打包、发货变成可执行任务 | 入库单、出库单、上架任务、拣货任务 | 作业结果必须与实物数量、库位、批次、责任人对应 |
| 调拨 | 在多仓之间转移库存，平衡供需 | 调拨单、调拨出库单、调拨入库单 | 调出、在途、调入必须可追踪，不允许只改一边库存 |
| 逆向退货 | 处理客户退货、拒收、退供和质检结果 | 售后单、退货入库单、退供应商单 | 退货质检结果决定是否回补可售库存 |

### 4.2 支撑域

支撑域为核心域提供稳定能力。

| 支撑域 | 业务目标 | 主要模型 |
| --- | --- | --- |
| 主数据 | 提供统一商品、仓库、供应商、客户、物流等发布语言 | SPU、SKU、供应商、客户/货主、仓库、库位、物流商 |
| 采购 | 管理采购需求、询价、价格、采购订单、到货跟进 | 采购申请、询价单、维价单、采购订单 |
| 供应商协同 | 让供应商参与准入、报价、接单、送货、对账 | 供应商档案、报价、订单确认、ASN、绩效评分 |
| 物流运输 | 选择物流渠道、获取面单、跟踪轨迹、沉淀费用 | 物流渠道、运单、轨迹、物流费用 |
| BMS 计费 | 根据业务事实生成费用明细、账单和对账结果 | 计费规则、费用明细、账单、对账单 |
| 财务 | 根据业务结算结果形成应收、应付、发票、收付款 | 应收单、应付单、发票、付款单、收款单 |

### 4.3 通用域

通用域不承载供应链差异化能力，应尽量使用稳定通用模型。

| 通用域 | 作用 |
| --- | --- |
| 权限 | 用户、角色、功能权限、数据权限、字段权限、单点登录 |
| 审计 | 操作日志、登录日志、敏感字段访问记录 |
| 字典 | 状态、枚举、业务类型、异常原因、审批原因配置 |
| 通知 | 站内信、邮件、短信、Webhook、待办 |
| 附件 | 资质证照、合同、发票、面单、质检图片 |
| OpenAPI | 外部应用、鉴权、签名、接口日志、回调、重试 |

## 5. 限界上下文总览

| 限界上下文 | 负责什么 | 不负责什么 | 拥有数据 | 主要生产事件 |
| --- | --- | --- | --- | --- |
| 主数据上下文 | 商品、供应商、客户/货主、仓库、库位、物流商等基础资料 | 采购执行、库存余额、仓库作业 | SPU、SKU、供应商、仓库、库位、物流商 | SKU已启用、供应商已启用、仓库已启用 |
| 供应商上下文 | 供应商准入、报价、订单确认、ASN、绩效 | 企业内部采购审批、仓库收货 | 供应商协同记录、报价、ASN、评分 | 供应商已准入、报价已提交、ASN已创建 |
| 采购上下文 | 采购申请、询价比价、采购订单、采购关单 | 实物收货、库存入账、财务付款 | 采购申请、采购订单、采购价格 | 采购订单已审核、采购订单已关闭 |
| OMS 上下文 | 销售订单接入、审核、拆合单、履约跟踪 | 仓内拣货、库存余额维护、物流实际配送 | 销售订单、履约单、发货指令 | 履约单已创建、订单已取消、订单已发货 |
| 中央库存上下文 | 可用库存、可售库存、预占、释放、扣减、流水 | 最细库位作业、商品主数据维护 | 库存账户、预占记录、库存流水 | 库存已预占、库存已释放、库存已扣减 |
| WMS 上下文 | 收货、验收、上架、拣货、复核、打包、发货、盘点 | 销售订单审核、全局库存承诺、费用结算 | 入库单、出库单、作业任务、库位库存 | 入库已上架、出库已发货、盘点已完成 |
| TMS 上下文 | 物流渠道、面单、运单、轨迹、物流异常 | 仓库打包、订单审核、客户退款 | 运单、面单、轨迹、物流费用原始数据 | 运单已创建、包裹已签收、物流异常已发生 |
| BMS 上下文 | 计费规则、费用明细、账单、对账 | 创造入库/出库/物流事实 | 计费规则、费用明细、账单 | 费用已生成、账单已确认 |
| 售后上下文 | 退款、退货、换货、补寄、拒收处理 | 仓库质检执行、财务实际打款 | 售后单、退款申请、补寄单 | 售后已审核、退货已入库、退款已确认 |
| 权限上下文 | 登录、用户、角色、权限、数据范围、操作日志 | 业务规则判断 | 用户、角色、权限、操作日志 | 用户已登录、权限已变更、操作已记录 |
| 集成上下文 | 外部应用、接口、Webhook、字段映射、重试 | 业务聚合内部规则 | 应用、凭证、接口日志、回调任务 | 回调已发送、接口调用失败 |

## 6. 子系统与业务主体

### 6.1 主数据系统

主数据是所有业务系统的共同语言。它负责创建、审核、启用、变更、停用基础资料，并把稳定版本分发给下游系统。

| 业务主体 | 作用 | 典型使用方 |
| --- | --- | --- |
| SPU/SKU | 定义卖什么、买什么、仓库识别什么 | 采购、OMS、库存、WMS、BMS |
| 类目/属性 | 支撑商品建档、筛选、仓储策略、计费规则 | 主数据、采购、WMS、BMS |
| 单位/包装 | 支撑采购单位、库存单位、销售单位和单位换算 | 采购、库存、WMS |
| 供应商 | 支撑采购寻源、合同、报价、送货、结算 | 供应商、采购、BMS |
| 客户/货主 | 支撑销售、仓储服务、多货主库存、费用归属 | OMS、WMS、BMS、权限 |
| 仓库/库区/库位 | 支撑库存归属、库内作业、分仓和计费 | WMS、库存、OMS |
| 物流商/渠道 | 支撑物流下单、面单、轨迹和费用 | TMS、OMS、WMS、BMS |

主流程：

```text
创建主数据
  -> 审核
  -> 启用
  -> 发布主数据事件
  -> 下游系统保存快照或映射
  -> 业务单据引用主数据版本
  -> 变更/停用时继续发布事件
```

### 6.2 采购系统

采购系统负责把业务需求转成对供应商的采购承诺。

| 业务主体 | 作用 |
| --- | --- |
| 采购申请 | 内部采购需求和审批载体 |
| 询价单/报价 | 支撑多供应商寻源、比价、议价 |
| 维价单 | 管控采购价格、阶梯价、账期和生效期 |
| 采购订单 | 正式向供应商采购的业务承诺 |
| 采购关单 | 结束未完成到货或后续履约可能 |

主流程：

```text
补货建议/人工需求
  -> 创建采购申请
  -> 审核
  -> 询价/比价/维价
  -> 创建采购订单
  -> 供应商确认
  -> 下发 WMS 入库指令
  -> 接收入库结果
  -> 采购关单/对账
```

### 6.3 供应商系统

供应商系统面向供应商协同，减少线下沟通和人工追踪。

| 业务主体 | 作用 |
| --- | --- |
| 供应商档案 | 维护供应商准入、资质、联系人、结算信息 |
| 供应商商品 | 维护供应商可供 SKU、最小起订量、交期 |
| 报价 | 支撑采购比价和价格维护 |
| 订单确认 | 供应商确认采购订单数量、价格、交期 |
| ASN/送货预约 | 供应商提前告知送货明细 |
| 绩效评分 | 根据质量、报价、交付时效、服务等形成采购参考 |

主流程：

```text
供应商注册/邀请
  -> 资质提交
  -> 准入审核
  -> 商品和报价维护
  -> 接收采购订单
  -> 确认订单/创建 ASN
  -> 仓库收货
  -> 对账/绩效评分
```

### 6.4 OMS 系统

OMS 负责把销售订单转成可执行的履约任务。

| 业务主体 | 作用 |
| --- | --- |
| 销售订单 | 来自渠道、门店、B2B、客户系统的订单 |
| 履约单 | OMS 内部履约聚合，承载审核、拆合、分仓、发货进度 |
| 分仓结果 | 决定由哪个仓或门店发货 |
| 库存预占请求 | 请求中央库存锁定可用量 |
| 发货指令 | 下发 WMS 执行出库作业 |

主流程：

```text
渠道订单进入
  -> 订单校验/审核
  -> 调用中央库存分仓和预占
  -> 拆单/合单
  -> 创建发货指令
  -> 下发 WMS
  -> 接收发货结果
  -> 回传渠道/客户
```

### 6.5 中央库存系统

中央库存系统负责统一库存可用性和库存承诺。

| 业务主体 | 作用 |
| --- | --- |
| 库存账户 | 按货主、仓库、SKU、批次等维度记录库存余额 |
| 库存预占 | 为订单、调拨、退供锁定可用库存 |
| 库存冻结 | 因质检、盘点、异常、临期等原因限制使用 |
| 库存流水 | 记录每一次库存变化的来源、数量和前后值 |
| 可售库存 | 面向 OMS 或销售渠道展示的库存结果 |

主流程：

```text
接收 WMS/外部仓库存事实
  -> 更新库存账户和流水
  -> 计算可用库存
  -> 计算可售库存
  -> 为订单/调拨/退供执行预占
  -> 根据发货/取消/异常扣减或释放
```

### 6.6 WMS 系统

WMS 负责仓库内部实物作业，是实物流执行系统。

| 业务主体 | 作用 |
| --- | --- |
| 入库单 | 承接采购入库、退货入库、调拨入库 |
| 收货/验收 | 记录到货数量、质检结果、批次效期 |
| 上架任务 | 指导货物进入指定库区库位 |
| 出库单 | 承接销售出库、调拨出库、退供应商出库 |
| 波次/拣货任务 | 指导人员从库位拿取指定商品 |
| 复核/打包/发货 | 校验商品、生成包裹并交接物流 |
| 盘点/调整/移库 | 维护库内库存准确性 |

入库流程：

```text
接收入库指令
  -> 到货登记
  -> 收货清点
  -> 质检
  -> 生成上架任务
  -> 上架确认
  -> 发布入库已上架事件
```

出库流程：

```text
接收出库指令
  -> 生成波次
  -> 分配库位和批次
  -> 生成拣货任务
  -> 拣货
  -> 复核
  -> 打包
  -> 发货交接
  -> 发布出库已发货事件
```

### 6.7 TMS 系统

TMS 负责物流服务选择、下单、面单、轨迹和物流费用基础数据。

| 业务主体 | 作用 |
| --- | --- |
| 物流商/渠道 | 定义可用物流服务 |
| 物流规则 | 按区域、重量、时效、成本匹配渠道 |
| 运单/面单 | 支撑仓库打印和物流跟踪 |
| 物流轨迹 | 记录揽收、在途、派送、签收、拒收、退回 |
| 物流费用 | 支撑预算、计费和对账 |

主流程：

```text
接收包裹和收件信息
  -> 匹配物流渠道
  -> 调用物流商下单
  -> 获取运单和面单
  -> WMS 打印面单并发货
  -> 订阅/拉取物流轨迹
  -> 回传 OMS/BMS
```

### 6.8 BMS 系统

BMS 负责把业务事实转成费用明细、账单和对账结果。

| 业务主体 | 作用 |
| --- | --- |
| 计费规则 | 定义按件、重量、体积、库龄、订单、公式等计费方式 |
| 报价方案 | 为不同客户、货主、物流商配置价格 |
| 计费事件 | 来自 WMS/TMS/库存的入库、出库、库存、物流事实 |
| 费用明细 | 每笔业务事实计算出的费用结果 |
| 账单/对账单 | 周期汇总并给客户、供应商、物流商确认 |

主流程：

```text
消费业务计费事件
  -> 匹配报价方案和计费规则
  -> 生成费用明细
  -> 周期汇总账单
  -> 对账确认
  -> 推送财务
```

### 6.9 售后与逆向系统

售后与逆向系统负责订单完成后的退货、退款、换货、补寄、拒收和退供应商。

| 业务主体 | 作用 |
| --- | --- |
| 售后单 | 记录退款、退货退款、换货、补寄等申请 |
| 退货单 | 指导客户或物流将商品退回 |
| 退货入库单 | WMS 收到退货后的入库处理 |
| 质检结果 | 决定良品上架、残次隔离、报废、退供 |
| 退供应商单 | 将不良品、滞销品或约定商品退回供应商 |

售后退货流程：

```text
客户发起售后
  -> 审核
  -> 创建退货任务
  -> WMS 收货质检
  -> 良品回补/残次隔离/报废
  -> 退款或换货
  -> 售后关闭
```

退供应商流程：

```text
发现需退供商品
  -> 创建退供应商单
  -> 中央库存预占退供库存
  -> WMS 拣货出库
  -> 发运给供应商
  -> 库存扣减
  -> 采购/SRM/财务处理
```

### 6.10 权限与审计系统

权限系统控制谁能登录、能看哪些页面、能操作哪些按钮、能访问哪些数据。

| 业务主体 | 作用 |
| --- | --- |
| 用户 | 系统操作者 |
| 角色 | 权限集合，如采购员、仓管员、财务、供应商、管理员 |
| 功能权限 | 菜单、页面、按钮、接口权限 |
| 数据权限 | 组织、仓库、货主、供应商、客户、单据归属范围 |
| 操作日志 | 记录关键写操作和责任人 |

主流程：

```text
创建用户/角色
  -> 分配功能权限和数据权限
  -> 用户登录
  -> 获取权限和 token
  -> 访问业务系统
  -> 写操作记录审计日志
```

## 7. 端到端主业务闭环

### 7.1 采购入库闭环

```text
主数据启用
  -> 供应商准入
  -> 采购询价/维价
  -> 采购订单审核
  -> 供应商确认/ASN
  -> WMS 收货验收
  -> WMS 上架
  -> 中央库存入账
  -> 采购进度更新
  -> BMS/财务生成应付或费用
```

关键状态变化：

| 聚合 | 典型状态变化 |
| --- | --- |
| 采购订单 | 草稿 -> 待审核 -> 已审核 -> 已确认 -> 部分入库/已入库 -> 已关闭 |
| 入库单 | 待收货 -> 收货中 -> 待上架 -> 已上架 -> 已完成 |
| 库存账户 | 可用库存增加，库存流水新增 |

关键事件：

| 事件 | 下游影响 |
| --- | --- |
| 采购订单已审核 | 通知供应商确认，准备生成入库指令 |
| ASN已创建 | WMS 可做预约收货 |
| 入库已上架 | 中央库存增加库存，采购更新到货进度，BMS 可计费 |

### 7.2 销售履约出库闭环

```text
销售订单进入 OMS
  -> OMS 审核
  -> 中央库存分仓/预占
  -> TMS 获取物流方案和面单
  -> OMS 下发出库指令
  -> WMS 拣货/复核/打包
  -> WMS 发货
  -> 中央库存扣减
  -> OMS 回传渠道
  -> TMS 跟踪轨迹
  -> BMS/财务结算
```

关键状态变化：

| 聚合 | 典型状态变化 |
| --- | --- |
| 履约单 | 待审核 -> 待预占 -> 已预占 -> 已下发 -> 已发货 -> 已完成 |
| 库存预占 | 已创建 -> 已确认 -> 已扣减，或已释放 |
| 出库单 | 待分配 -> 待拣货 -> 待复核 -> 待打包 -> 已发货 |

关键事件：

| 事件 | 下游影响 |
| --- | --- |
| 库存已预占 | OMS 可以下发 WMS |
| 出库已发货 | OMS 回传渠道，中央库存扣减，BMS 生成操作费 |
| 物流已签收 | OMS 完成履约，售后期开始或结束 |

### 7.3 调拨闭环

```text
发现库存不均或补货需求
  -> 创建调拨单
  -> 调出仓库存预占
  -> 调出仓 WMS 出库
  -> 形成在途库存
  -> 调入仓 WMS 收货上架
  -> 调出仓扣减
  -> 调入仓增加
  -> 调拨关闭
```

关键不变量：

- 调拨必须同时表达调出、在途、调入三个阶段。
- 调出仓发货数量和调入仓收货数量可能不一致，必须能记录差异。
- 调拨不能只改库存余额，必须有调拨单和库存流水。

### 7.4 售后退货闭环

```text
客户发起售后
  -> 售后审核
  -> 创建退货任务
  -> 退货物流回仓
  -> WMS 收货质检
  -> 良品上架/残次隔离/报废
  -> 中央库存按质检结果处理
  -> 财务退款或调整
  -> 售后关闭
```

关键不变量：

- 未质检的退货不能直接回补可售库存。
- 良品、残次、报废必须进入不同库存状态或库区。
- 退款、库存回补、售后关闭要有可追溯依据。

### 7.5 供应商退货闭环

```text
采购/质检/库存发现需退供
  -> 创建退供应商单
  -> 审核
  -> 中央库存预占退供库存
  -> WMS 拣货出库
  -> 发运给供应商
  -> 中央库存扣减
  -> 采购/SRM 记录退供
  -> 财务冲减应付或生成应收
```

关键不变量：

- 退供商品必须先锁定，避免被销售或调拨占用。
- 已发运退供后，库存应从企业可用库存中扣减。
- 退供要关联采购、供应商、质检或库存异常来源。

## 8. 跨系统交互方式

### 8.1 命令与事件

供应链系统中的跨系统交互分两类：

| 类型 | 含义 | 示例 | 设计要求 |
| --- | --- | --- | --- |
| 命令 | 请求某个上下文执行动作 | 创建采购订单、请求库存预占、下发出库单 | 必须有发起方、目标方、幂等键、校验规则、失败返回 |
| 领域事件 | 表示某个业务事实已经发生 | 入库已上架、库存已扣减、出库已发货 | 必须有事件 ID、版本、发生时间、来源聚合、消费者、重试机制 |

典型链路：

```text
OMS 发送“请求库存预占”命令
  -> 中央库存处理库存预占聚合
  -> 中央库存发布“库存已预占”事件
  -> OMS 消费事件并推进履约单
  -> OMS 发送“下发出库单”命令
  -> WMS 处理出库单聚合
  -> WMS 发布“出库已发货”事件
  -> OMS、中央库存、BMS 分别消费
```

### 8.2 上下文依赖关系

| 上游 | 下游 | 协作方式 | 说明 |
| --- | --- | --- | --- |
| 主数据 | 全部业务系统 | 发布语言、主数据事件、查询接口 | 下游保存必要快照，避免历史单据被主数据变更污染 |
| 采购 | 供应商 | 命令/API | 采购订单、询价、变更需要供应商确认 |
| 供应商 | 采购/WMS | 事件/API | 报价提交、订单确认、ASN 创建 |
| OMS | 中央库存 | 命令/API | 查询、分仓、预占、释放 |
| 中央库存 | OMS/渠道 | 事件/同步 | 可售库存变化、预占结果、释放结果 |
| OMS | WMS | 命令/API | 下发出库、取消出库、变更出库 |
| WMS | OMS/库存/BMS | 事件 | 发货、入库、盘点、调整、异常 |
| WMS/TMS | BMS | 事件/计费明细源 | BMS 只根据业务事实计费，不创造作业事实 |
| BMS | 财务 | 命令/API | 推送应收、应付、账单确认结果 |
| 权限 | 全部业务系统 | token 校验、权限查询、审计日志 | 统一登录和授权，业务系统做本地缓存 |

### 8.3 一致性与补偿原则

| 场景 | 设计原则 |
| --- | --- |
| 重复请求 | 使用业务单号、幂等键或事件 ID 去重 |
| 预占失败 | OMS 进入缺货、换仓、拆单、人工处理或取消流程 |
| WMS 短拣 | WMS 发布短拣事件，OMS 决定部分发货、换仓、取消，库存释放差异 |
| 发货回传失败 | WMS 保留发货事实，集成层重试，OMS 支持补偿查询 |
| 库存扣减失败 | 中央库存进入异常流水，触发人工处理和对账 |
| 费用重算 | BMS 使用计费事件版本和业务单号防重复，保留调整/红冲记录 |
| 主数据变更 | 历史单据保留快照，新单据使用新版本 |

## 9. 第一版建设蓝图

如果先做一套能跑通核心业务的供应链系统，建议按以下顺序建设：

| 阶段 | 建设重点 | 目标 |
| --- | --- | --- |
| 第 1 阶段 | 主数据、采购、基础库存、简单入库/出库 | 跑通进销存闭环 |
| 第 2 阶段 | WMS 入库、出库、库位、盘点、调拨 | 仓库作业可执行、可追踪 |
| 第 3 阶段 | OMS、中央库存、分仓预占、一盘货 | 支撑多渠道订单履约，避免超卖 |
| 第 4 阶段 | 供应商协同、采购对账、供应商评分 | 提升采购协同效率 |
| 第 5 阶段 | TMS、物流面单、轨迹、物流规则 | 完善发货和配送跟踪 |
| 第 6 阶段 | BMS、财务结算、对账、报表 | 完成经营和费用闭环 |
| 第 7 阶段 | OpenAPI、权限审计、监控预警 | 支撑外部系统接入和规模化运营 |

## 11. 当前结论

供应链系统不是一个单独应用，而是一组围绕商品流转、库存承诺、仓储执行、订单履约和费用结算协作的限界上下文。第一版设计应以“主数据 + 采购/供应商 + OMS + 中央库存 + WMS + BMS + 权限”为核心，再逐步扩展 TMS、财务、OpenAPI 和报表分析。
