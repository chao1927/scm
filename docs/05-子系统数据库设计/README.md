## 维护规则

- `03-核心业务模型` 负责聚合、不变量、命令、事件和状态机，不维护数据库字段清单。
- `04-子系统功能设计` 负责页面、流程、权限、枚举和操作场景，不维护数据库字段清单。
- `05-子系统数据库设计` 负责表、字段类型、枚举落库、索引、事件表、操作日志表和后续 DDL。
- 字段新增时，先判断它是领域不变量、页面展示字段、读模型字段还是技术字段，再决定是否进入业务表。
