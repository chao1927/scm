### 1.2 命名约定

| 类型 | 示例 | 说明 |
| --- | --- | --- |
| Controller | `IamUserController` | 前端 HTTP 接口 |
| OpenApiController | `IamPermissionOpenApiController` | 业务系统权限校验入口 |
| QueryService | `IamPermissionQueryService` | 查询权限快照和读模型 |
| ApplicationService | `IamUserApplicationService` | 编排权限、幂等、事务、权限版本和事件 |

## 2. 登录、个人中心和当前权限接口

### 2.1 用户登录

`POST /api/iam/v1/auth/login`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `IamAuthController.login(LoginRequest)` 接收账号、密码、验证码、MFA、应用和客户端信息。 |
| 应用层 | `AuthApplicationService.login(command)` 校验应用启用、账号状态、密码策略、MFA、安全策略和登录频率。 |
| 领域层 | `UserAggregate.loginSuccess/loginFailed` 更新失败次数、锁定状态；`UserTokenAggregate.issue` 签发会话。 |
| 基础设施层 | 密码组件校验 hash，JWT 组件签名，Redis 写 Token JTI 和权限快照，写登录日志。 |
| 事件/返回 | 成功返回 Token、用户、菜单、权限、数据范围；生产 `UserLoggedIn`。失败写 `LoginFailedRecorded`。 |
| 异常 | 密码错误返回 `401`；需 MFA 返回 `422 MFA_REQUIRED`；失败过多返回 `429` 并锁定。 |

### 2.2 刷新 Token 和退出登录

`POST /api/iam/v1/auth/refresh`、`POST /api/iam/v1/auth/logout`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | 解析 refresh token 或当前 access token。 |
| 应用层 | 校验 Token JTI、用户状态、应用状态和权限版本；登出按当前会话幂等撤销。 |
| 领域层 | `UserTokenAggregate.refresh/invalidate` 推进 Token 状态。 |
| 基础设施层 | 更新 `iam_user_token`，刷新 Redis 会话，写审计。 |
| 事件/返回 | 刷新返回新 Token；登出返回 `204`；事件为 `TokenRefreshed`、`TokenInvalidated`。 |
| 异常 | refresh token 过期或已撤销返回 `401 TOKEN_REVOKED`。 |

### 2.3 获取当前用户、维护个人资料和权限快照

`GET /api/iam/v1/users/me`、`PUT /api/iam/v1/users/me/profile`、`POST /api/iam/v1/users/me/password`、`GET /api/iam/v1/users/me/permissions`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | 从 Token 解析用户和应用编码；个人资料接口接收昵称、手机号、邮箱；改密接口接收旧密码、新密码和 MFA 校验。 |
| 应用层 | `CurrentUserQueryService` 校验会话状态，读取或重建权限快照；`CurrentUserApplicationService` 校验本人操作、密码策略和幂等。 |
| 领域层 | 查询无聚合行为；个人资料和改密由用户聚合校验用户状态、资料可改字段和密码强度。 |
| 基础设施层 | 查询用户、角色、菜单、权限、数据范围；优先读取 Redis 权限快照；资料变更写用户表、审计表并刷新用户缓存。 |
| 返回/事件 | 查询返回用户资料、角色、菜单树、按钮/API/字段权限、数据范围、权限版本；资料或密码变更产生 `UserProfileChanged`、`PasswordChanged` 和审计日志。 |
| 异常 | 权限版本过期时强制重建快照；用户停用返回 `401`；旧密码错误返回 `401`；密码不符合策略返回 `422`。 |

## 3. 用户、角色和授权接口

### 3.1 用户查询、新增、修改和分配角色

`GET /api/iam/v1/users`、`GET /api/iam/v1/users/{userId}`、`POST /api/iam/v1/users`、`PUT /api/iam/v1/users/{userId}`、`POST /api/iam/v1/users/{userId}/roles`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `IamUserController` 接收账号、姓名、用户类型、组织、外部主体、分页、用户ID和待分配角色集合。 |
| 应用层 | 查询校验 `iam:user:read`；写命令校验 `iam:user:create/update/role`、幂等、账号唯一、角色有效性和数据范围。 |
| 领域层 | `UserAggregate.create/changeProfile/assignRoles` 校验用户类型、MFA、手机号邮箱唯一、角色生效期和状态约束。 |
| 基础设施层 | 保存 `iam_user`、`iam_user_role`，可调用主数据校验供应商/客户/货主绑定，写事件和审计。 |
| 事件/返回 | `UserCreated`、`UserRoleAssigned`；返回用户详情、角色、数据范围和状态时间线。 |
| 异常 | 账号重复返回 `409 DUPLICATE_USERNAME`；外部主体未启用返回 `422`。 |

### 3.2 启用、停用、锁定和重置密码

`POST /api/iam/v1/users/{userId}/enable`、`POST /api/iam/v1/users/{userId}/disable`、`POST /api/iam/v1/users/{userId}/lock`、`POST /api/iam/v1/users/{userId}/reset-password`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | 接收原因、锁定到期时间、初始密码策略和版本。 |
| 应用层 | 校验按钮权限、幂等、乐观锁；停用用户时加载有效会话。 |
| 领域层 | 用户聚合推进待激活、已启用、已锁定、已停用状态；密码策略服务生成或校验新密码。 |
| 基础设施层 | 更新 `iam_user`，撤销或标记 Token，写审计和事件。 |
| 事件/返回 | `UserActivated`、`UserLocked`、`UserDisabled`、`TokenInvalidated`。 |
| 异常 | 停用最后一个系统管理员返回 `422 BUSINESS_RULE_FAILED`。 |

### 3.3 角色查询、新增、修改、启停和角色用户

`GET /api/iam/v1/roles`、`GET /api/iam/v1/roles/{roleId}`、`POST /api/iam/v1/roles`、`PUT /api/iam/v1/roles/{roleId}`、`POST /api/iam/v1/roles/{roleId}/enable`、`POST /api/iam/v1/roles/{roleId}/disable`、`GET /api/iam/v1/roles/{roleId}/users`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `IamRoleController` 接收角色编码、名称、类型、应用范围、默认数据范围；角色用户接口接收用户筛选和分页。 |
| 应用层 | 查询校验 `iam:role:read/user`；写命令校验 `iam:role:*`、角色编码唯一、系统角色保护和乐观锁。 |
| 领域层 | 查询无聚合行为；写命令由 `RoleAggregate` 校验角色类型、应用范围和停用前影响用户。 |
| 基础设施层 | 保存 `iam_role`，停用角色时提升相关用户权限版本并可撤销会话；角色用户查询 `iam_user_role` 和用户读模型。 |
| 事件/返回 | `RoleCreated`、`RoleEnabled`、`RoleDisabled`、`TokenInvalidated` 可选；返回角色详情和角色下用户分页。 |
| 异常 | 系统内置角色禁止停用返回 `422`。 |

### 3.4 角色授权和权限变更记录

`GET /api/iam/v1/roles/{roleId}/permission-tree`、`POST /api/iam/v1/roles/{roleId}/permissions`、`POST /api/iam/v1/roles/{roleId}/permissions/revoke`、`GET /api/iam/v1/roles/{roleId}/permission-changes`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `IamRolePermissionController` 接收角色、应用、权限点集合、授权原因和变更记录查询条件。 |
| 应用层 | 授权矩阵和变更记录校验 `iam:role:read`；写命令校验 `iam:role:page/revoke`、角色启用、权限点启用、幂等和权限版本。 |
| 领域层 | 查询无聚合行为；写命令由 `RolePermissionAggregate.grant/revoke` 计算新增和取消授权，保护系统角色权限边界。 |
| 基础设施层 | 批量写 `iam_role_permission`，提升权限版本，清理 Redis 权限缓存；变更记录查询授权审计表。 |
| 事件/返回 | `RolePermissionGranted`、`RolePermissionRevoked`；返回授权数量、权限版本和变更前后差异。 |
| 异常 | 授权已停用权限点返回 `422 PERMISSION_NOT_ENABLED`。 |

### 3.5 用户角色和数据权限

`GET /api/iam/v1/user-roles`、`POST /api/iam/v1/user-roles`、`POST /api/iam/v1/user-roles/{userRoleId}/revoke`、`GET /api/iam/v1/data-scopes`、`POST /api/iam/v1/data-scopes`、`POST /api/iam/v1/data-scopes/{dataScopeId}/disable`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `IamUserRoleController`、`IamDataScopeController` 接收用户、角色、主体类型、资源类型和资源ID。 |
| 应用层 | 校验授权权限、用户/角色状态、主数据资源是否启用、幂等和乐观锁。 |
| 领域层 | 用户角色聚合保证有效期不冲突；数据权限聚合校验范围类型、资源类型匹配和停用状态机。 |
| 基础设施层 | 写 `iam_user_role`、`iam_data_scope`，刷新权限快照和数据范围缓存。 |
| 事件/返回 | `UserRoleAssigned`、`UserRoleRevoked`、`DataScopeChanged`。 |
| 异常 | 自定义范围但资源为空返回 `400`；资源未启用返回 `422`。 |

## 4. 菜单、权限点、应用和SSO接口

### 4.1 菜单页面维护

`GET /api/iam/v1/menus`、`POST /api/iam/v1/menus`、`PUT /api/iam/v1/menus/{menuId}`、`POST /api/iam/v1/menus/sort`、`POST /api/iam/v1/menus/{menuId}/disable`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | 接收应用、菜单编码、类型、路由、组件、排序、可见性。 |
| 应用层 | 校验菜单权限、应用启用、菜单编码应用内唯一和树结构合法。 |
| 领域层 | 菜单页面聚合保证目录/页面/外链字段约束和排序不变量。 |
| 基础设施层 | 保存 `iam_menu_page`，刷新菜单缓存，写审计。 |
| 事件/返回 | 菜单变化通过 `PermissionChanged` 或权限缓存失效事件通知业务系统。 |
| 异常 | 菜单存在子节点时停用需级联确认，否则返回 `422`。 |

### 4.2 权限点维护、绑定资源和停用

`GET /api/iam/v1/permissions`、`GET /api/iam/v1/permissions/{permissionId}`、`POST /api/iam/v1/permissions`、`PUT /api/iam/v1/permissions/{permissionId}`、`POST /api/iam/v1/permissions/{permissionId}/bind-menu`、`POST /api/iam/v1/permissions/{permissionId}/bind-api`、`POST /api/iam/v1/permissions/{permissionId}/disable`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `IamPermissionController` 接收权限编码、资源类型、动作类型、菜单ID、API方法路径、字段编码和停用原因。 |
| 应用层 | 查询校验 `iam:permission:read`；写命令校验权限点管理权限、编码全局唯一、菜单存在、API路径格式和方法合法。 |
| 领域层 | 权限点聚合校验菜单/按钮/API/字段类型与资源字段匹配，停用前计算已授权角色影响。 |
| 基础设施层 | 保存 `iam_permission`，刷新网关权限映射缓存。 |
| 事件/返回 | `PermissionRegistered`、`PermissionChanged`、`PermissionApiBound`、`PermissionDisabled`；绑定菜单时返回菜单资源摘要。 |
| 异常 | API路径重复绑定不同权限点返回 `409`。 |

### 4.3 应用、SSO和枚举配置

`GET /api/iam/v1/apps`、`POST /api/iam/v1/apps`、`PUT /api/iam/v1/apps/{appId}`、`POST /api/iam/v1/apps/{appId}/toggle`、`GET /api/iam/v1/sso-clients`、`POST /api/iam/v1/sso-clients`、`POST /api/iam/v1/sso-clients/{ssoId}/reset-secret`、`GET /api/iam/v1/enums`、`POST /api/iam/v1/enums`、`PUT /api/iam/v1/enums/{enumItemId}`、`POST /api/iam/v1/enums/{enumItemId}/disable`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | 接收应用编码、类型、首页地址、启停状态、SSO回调、授权类型、Token有效期和枚举项。 |
| 应用层 | 校验系统配置权限、应用编码唯一、回调地址白名单、密钥重置高危权限和枚举非核心可变规则。 |
| 领域层 | 应用聚合和 SSO 客户端聚合校验启停状态和密钥生命周期；枚举规则服务校验枚举值唯一。 |
| 基础设施层 | 保存 `iam_app`、`iam_sso`、枚举配置，客户端密钥只返回一次明文，后续只存 hash。 |
| 事件/返回 | `IamAppCreated/Enabled/Disabled`、`SsoClientConfigured`、`EnumItemChanged`。 |
| 异常 | 回调地址非法返回 `400`；停用应用时有在线会话需同步失效。 |

## 5. 会话、日志、OpenAPI和事件入口

### 5.1 会话管理

`GET /api/iam/v1/sessions`、`GET /api/iam/v1/sessions/{userTokenId}`、`POST /api/iam/v1/sessions/{userTokenId}/kick-out`、`POST /api/iam/v1/sessions/kick-out`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | 接收用户、应用、Token状态、登录时间、会话ID集合和踢出原因。 |
| 应用层 | 查询校验会话读权限；单个/批量踢出校验高危按钮权限、幂等和会话状态。 |
| 领域层 | `UserTokenAggregate.kickOut` 推进 Token 状态为撤销，批量场景逐个聚合处理并汇总结果。 |
| 基础设施层 | 更新 `iam_user_token`，删除 Redis 会话，写审计。 |
| 事件/返回 | `SessionKickedOut`、`TokenInvalidated`；批量返回成功数、失败数和失败明细。 |
| 异常 | 已过期会话重复踢出幂等返回成功。 |

### 5.2 登录日志和操作日志

`GET /api/iam/v1/login-logs`、`GET /api/iam/v1/login-logs/{loginLogId}`、`GET /api/iam/v1/operation-logs`、`GET /api/iam/v1/operation-logs/{operationLogId}`、`POST /api/iam/v1/operation-logs/export`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | 接收账号、结果、应用、模块、操作类型、时间范围、详情ID和分页。 |
| 应用层 | 校验审计权限和数据范围；详情按敏感字段脱敏；导出创建异步任务并记录审计。 |
| 领域层 | 查询不触发聚合行为；敏感日志标记由审计服务处理。 |
| 基础设施层 | 查询 `iam_login_log`、`iam_operation_log`、`iam_operation_audit_log`，前后快照脱敏。 |
| 返回 | 登录日志、登录失败原因、操作日志、前后快照、导出任务号。 |
| 异常 | 时间跨度过大返回 `400`；导出量过大异步处理。 |

### 5.3 Token校验、权限快照、权限检查和数据范围解析

`POST /openapi/iam/v1/tokens/validate`、`GET /openapi/iam/v1/users/me/permissions`、`POST /openapi/iam/v1/permissions/check`、`POST /openapi/iam/v1/data-scopes/resolve`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `IamPermissionOpenApiController` 校验来源系统、签名、Token、应用、权限点和数据范围解析参数。 |
| 应用层 | Token 校验服务校验签名、JTI、状态、用户状态、应用状态；权限快照服务返回菜单/按钮/API/字段权限；权限检查服务匹配角色、权限点、API和数据范围。 |
| 领域层 | 无业务状态推进；拒绝访问通过审计服务形成审计事实。 |
| 基础设施层 | 读取 Redis 权限快照，未命中查用户/角色/权限/数据范围表重建。 |
| 返回 | 用户身份、菜单/按钮/API/字段权限、allowed/denyReason、数据范围过滤表达式和权限版本。 |
| 异常 | Token撤销返回 `401`；权限拒绝返回 `403` 并写 `PermissionCheckDenied`。 |

### 5.4 操作日志写入和审批任务

`POST /openapi/iam/v1/operation-logs`、`POST /openapi/iam/v1/approval-tasks`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | 接收来源系统、幂等键、操作人、权限点、业务对象、审批业务载荷。 |
| 应用层 | 操作日志按 `sourceSystem + requestId + operationType + bizNo` 幂等；审批任务按来源单据和版本幂等。 |
| 领域层 | 操作日志聚合记录审计事实；审批实例聚合创建节点和待办。 |
| 基础设施层 | 写 `iam_operation_log`、审计表、审批实例/任务表和事件表。 |
| 事件/返回 | `AuditLogCreated`、`ApprovalStarted`；返回日志ID或审批实例ID。 |
| 异常 | 审批配置缺失返回 `422`；日志重复写入返回历史结果。 |

### 5.5 通用事件入口

`POST /internal/iam/v1/events`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `IamEventConsumer.onMessage` 解析事件信封、来源系统和事件类型。 |
| 应用层 | `IamEventDispatchService` 先写 `iam_event_consume_log`，再分派主数据资源、人事入离职、API扫描、敏感操作和审批补偿处理器。 |
| 领域层 | 根据事件类型创建/停用用户、更新可授权资源、生成权限点建议或写审计。 |
| 基础设施层 | 保存消费日志、业务状态、审计和必要的权限缓存失效标记。 |
| 返回 | `consumeId`、`consumeStatus`、`idempotentHit`。 |
| 异常 | 不支持事件类型标记已忽略；可重试业务异常保留失败原因。 |
