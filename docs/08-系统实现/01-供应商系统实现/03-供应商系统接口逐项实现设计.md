# 03-供应商系统接口逐项实现设计

> 本文承接 [02-供应商系统接口事件实现逻辑](02-供应商系统接口事件实现逻辑.md)，继续把 [01-供应商系统接口设计](01-供应商系统接口设计.md) 中的每个接口拆成可编码的实现小节。每个接口都按“接口层 -> 应用层 -> 领域层 -> 基础设施层 -> 事件/事务 -> 异常处理”的结构说明，方便后续直接落 Java Spring Boot + Dubbo 代码。

## 1. 统一代码分层

### 1.1 包结构

```text
supplier-service
  ├── interfaces
  │   ├── web              # 前端 HTTP Controller
  │   ├── openapi          # 跨系统 OpenAPI Controller
  │   ├── mq               # RocketMQ Listener
  │   └── job              # Outbox/补偿/导出 Job
  ├── application
  │   ├── command          # 写命令应用服务
  │   ├── query            # 查询服务
  │   ├── event            # 外部事件消费应用服务
  │   └── dto              # Command/Query/Result DTO
  ├── domain
  │   ├── supplier         # 供应商聚合
  │   ├── item             # 供应商商品聚合
  │   ├── po               # 采购订单确认聚合
  │   ├── asn              # ASN 聚合
  │   ├── returnorder      # 退供应商单聚合
  │   ├── reconcile        # 对账确认聚合
  │   ├── quality          # 质量问题聚合
  │   ├── score            # 供应商评分聚合
  │   └── shared           # 值对象、领域事件、异常
  └── infrastructure
      ├── persistence      # Repository 实现、MyBatis Mapper
      ├── rpc              # 采购/WMS/TMS/BMS/主数据/权限客户端
      ├── mq               # 事件发布
      ├── idempotency      # 幂等组件
      ├── audit            # 操作审计
      └── file             # 附件、导入导出
```

### 1.2 命名约定

| 类型 | 示例 | 说明 |
| --- | --- | --- |
| Controller | `SupplierAsnController` | 只做协议转换和基础参数校验 |
| QueryService | `SupplierAsnQueryService` | 只查读模型，可调用外部查询 RPC |
| ApplicationService | `SupplierAsnApplicationService` | 编排权限、幂等、事务、聚合、事件、审计 |
| Aggregate | `AsnAggregate` | 保护状态机和不变量 |
| Repository | `AsnRepository` | 保存/加载聚合，不暴露表结构给领域层 |
| Mapper | `SupAsnMapper` | MyBatis 数据访问 |
| DomainEvent | `AsnSubmittedEvent` | 领域事实，写入 `sup_domain_event` |
| EventHandler | `PurchaseOrderPublishedHandler` | 外部事件消费处理器 |

### 1.3 写接口标准模板

```text
Controller
  1. 接收请求头、路径参数、请求体
  2. 校验基础格式
  3. 转换 Command
  4. 调用 ApplicationService

ApplicationService
  1. 校验 token、按钮权限、数据权限
  2. 校验 X-Idempotency-Key
  3. 加载聚合和必要快照
  4. 调用聚合行为或领域服务
  5. 同一事务保存业务表、事件表、审计表
  6. 返回命令结果

Domain
  1. 校验状态机
  2. 校验业务不变量
  3. 修改聚合状态
  4. 产生领域事件

Infrastructure
  1. Repository 保存聚合
  2. Outbox 保存事件
  3. Audit 保存操作日志
  4. RPC/文件/缓存适配
```

### 1.4 查询接口标准模板

```text
Controller
  1. 接收查询条件
  2. 转换 Query
  3. 调用 QueryService

QueryService
  1. 校验菜单权限和数据范围
  2. 追加 supplierId/orgId/dataScope 查询条件
  3. 查询 MySQL 读模型
  4. 必要时调用外部 RPC 补充快照
  5. 返回分页、详情或统计 DTO
```

## 2. 工作台接口

### 2.1 查询工作台统计

`GET /api/supplier/v1/workbench/summary`

| 层     | 实现逻辑                                                                                                   |
| ----- | ------------------------------------------------------------------------------------------------------ |
| 接口层   | `SupplierWorkbenchController.summary(WorkbenchSummaryQueryRequest)` 解析 `supplierId`、时间范围和当前用户。         |
| 应用层   | `SupplierWorkbenchQueryService.querySummary(query)` 校验 `supplier:workbench:read`，按用户类型追加供应商/组织数据范围。    |
| 领域层   | 无聚合行为；这是读模型查询，不触发领域事件。                                                                                 |
| 基础设施层 | `WorkbenchQueryMapper` 聚合查询 `sup_order`、`sup_asn`、`sup_supplier_return`、`sup_reconcile`、`sup_quality`。 |
| 返回    | 待确认订单数、待发货 ASN 数、待处理退供数、待确认对账数、质量问题数。                                                                  |
| 异常    | 无权限返回 `403`；查询超时返回 `500`；外部供应商无绑定供应商返回空统计。                                                             |

### 2.2 查询待办列表

`GET /api/supplier/v1/workbench/todos`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `SupplierWorkbenchController.todos(TodoPageQueryRequest)` 接收 `todoType/pageNo/pageSize`。 |
| 应用层 | `SupplierTodoQueryService.pageTodos(query)` 校验工作台读权限和数据范围。 |
| 领域层 | 无聚合行为。 |
| 基础设施层 | 从订单确认、ASN、退供、对账、质量整改读模型 union 或统一待办表查询。 |
| 返回 | `todoId`、`businessType`、`businessNo`、`title`、`statusName`、`targetRoute`。 |
| 异常 | `pageSize` 超限制返回 `400`；数据范围为空返回空分页。 |

## 3. 供应商档案接口

### 3.1 查询供应商档案

`GET /api/supplier/v1/profile`

| 层     | 实现逻辑                                                                                      |
| ----- | ----------------------------------------------------------------------------------------- |
| 接口层   | `SupplierProfileController.profile(ProfileQueryRequest)` 获取当前用户绑定供应商；内部用户可传 `supplierId`。 |
| 应用层   | `SupplierProfileQueryService.getProfile(query)` 校验 `supplier:profile:read` 和供应商数据权限。      |
| 领域层   | 无聚合行为。                                                                                    |
| 基础设施层 | 优先查本地供应商快照；必要时调用 `MdmSupplierRpcClient.getSupplier(supplierId)` 补充主数据最新资料。                |
| 返回    | 供应商编码、名称、状态、联系人、资质、结算资料、风险等级、快照版本。                                                        |
| 异常    | 主数据 RPC 失败时返回本地快照并标记 `profileFresh=false`；无供应商范围返回 `403`。                                 |

### 3.2 查询资料变更记录

`GET /api/supplier/v1/profile/change-requests`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `SupplierProfileController.pageChangeRequests(ChangeRequestPageQuery)` 接收状态、时间、分页。 |
| 应用层 | `SupplierProfileQueryService.pageChangeRequests(query)` 校验档案读权限。 |
| 领域层 | 无聚合行为。 |
| 基础设施层 | 查询供应商资料变更记录表或变更读模型，按 `supplierId + created_at` 排序。 |
| 返回 | 变更单号、变更字段摘要、状态、申请人、申请时间、审批结果。 |
| 异常 | 查询条件时间跨度过大返回 `400`，建议前端限制一年内。 |

### 3.3 申请资料变更

`POST /api/supplier/v1/profile/change-requests`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `SupplierProfileController.submitChange(ProfileChangeSubmitRequest)` 校验请求体和 `X-Idempotency-Key`。 |
| 应用层 | `SupplierProfileApplicationService.submitChange(command)` 校验 `supplier:supplier_profile:change`、供应商绑定、幂等和档案版本。 |
| 领域层 | `SupplierAggregate.submitProfileChange(...)` 校验字段是否允许变更、关键字段是否需要审批、附件是否满足资料要求。 |
| 基础设施层 | `SupplierRepository` 保存变更申请；`DomainEventRepository` 写 `SupplierProfileChangeSubmitted`；`AuditLogRepository` 写审计。 |
| 事件 | 生产 `SupplierProfileChangeSubmitted`，主数据/采购/BMS 可消费。 |
| 异常 | 字段不可变返回 `422`；版本冲突返回 `409`；同幂等键请求内容不同返回 `409 IDEMPOTENCY_CONFLICT`。 |

### 3.4 撤回资料变更

`POST /api/supplier/v1/profile/change-requests/{changeRequestId}/withdraw`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `SupplierProfileController.withdrawChange(changeRequestId, request)` 接收撤回原因和版本。 |
| 应用层 | `SupplierProfileApplicationService.withdrawChange(command)` 校验权限、申请人或管理员权限、幂等。 |
| 领域层 | `SupplierAggregate.withdrawProfileChange(...)` 只允许待审批状态撤回。 |
| 基础设施层 | 更新变更申请状态为已撤回，写事件和审计。 |
| 事件 | `SupplierProfileChangeWithdrawn`，用于关闭审批待办。 |
| 异常 | 已审批/已生效不可撤回，返回 `409 STATE_CONFLICT`。 |

## 4. 供应商账号绑定接口

### 4.1 查询绑定列表

`GET /api/supplier/v1/user-bindings`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `SupplierUserBindingController.page(UserBindingPageQuery)` 接收 `supplierId/keyword/bindingRole/status`。 |
| 应用层 | `SupplierUserBindingQueryService.page(query)` 校验 `supplier:user_binding:read` 和数据范围。 |
| 领域层 | 无聚合行为。 |
| 基础设施层 | 查询 `sup_supplier_user`，必要时批量调用权限系统补充用户名、手机号、账号状态。 |
| 返回 | 绑定用户、角色、主账号标识、启停状态、绑定时间。 |
| 异常 | 权限系统超时返回本地快照和 `userFresh=false`。 |

### 4.2 查询绑定详情

`GET /api/supplier/v1/user-bindings/{supplierUserId}`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `SupplierUserBindingController.detail(supplierUserId)`。 |
| 应用层 | `SupplierUserBindingQueryService.getDetail(id)` 校验读权限和该绑定所属供应商范围。 |
| 领域层 | 无聚合行为。 |
| 基础设施层 | 查询 `sup_supplier_user`，调用 IAM 查询用户详情和角色摘要。 |
| 返回 | 绑定详情、权限角色、操作日志摘要。 |
| 异常 | 记录不存在或无数据权限返回 `404`，避免泄露其他供应商数据。 |

### 4.3 绑定供应商用户

`POST /api/supplier/v1/user-bindings`

| 层     | 实现逻辑                                                                                                 |
| ----- | ---------------------------------------------------------------------------------------------------- |
| 接口层   | `SupplierUserBindingController.bind(BindSupplierUserRequest)`。                                       |
| 应用层   | `SupplierUserBindingApplicationService.bind(command)` 校验 `supplier:user_binding:bind`、供应商状态、用户存在、幂等。 |
| 领域层   | `SupplierAggregate.bindUser(...)` 校验同一用户不可重复绑定同一供应商，主账号唯一。                                           |
| 基础设施层 | `SupplierUserRepository` 插入 `sup_supplier_user`；调用 IAM 可选同步供应商门户角色；写事件和审计。                           |
| 事件    | `SupplierUserBound`。                                                                                 |
| 异常    | 用户不存在 `422`；重复绑定 `409`；供应商停用 `422`；IAM 同步失败生成同步待办。                                                   |

### 4.4 解绑供应商用户

`POST /api/supplier/v1/user-bindings/{supplierUserId}/unbind`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `SupplierUserBindingController.unbind(id, request)`。 |
| 应用层 | `SupplierUserBindingApplicationService.unbind(command)` 校验解绑权限、幂等、版本。 |
| 领域层 | `SupplierAggregate.unbindUser(...)` 校验最后一个主账号不能直接解绑，除非先指定新主账号。 |
| 基础设施层 | 更新 `sup_supplier_user.status/unbound_at`，写事件和审计，调用 IAM 取消供应商门户角色。 |
| 事件 | `SupplierUserUnbound`。 |
| 异常 | 最后主账号解绑返回 `422 BUSINESS_RULE_FAILED`；版本冲突返回 `409`。 |

### 4.5 启用绑定

`POST /api/supplier/v1/user-bindings/{supplierUserId}/enable`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `SupplierUserBindingController.enable(id, request)`。 |
| 应用层 | `SupplierUserBindingApplicationService.enable(command)` 校验 `supplier:supplier:enable` 和供应商可协同状态。 |
| 领域层 | `SupplierAggregate.enableUserBinding(...)` 校验当前状态为停用。 |
| 基础设施层 | 更新 `sup_supplier_user.status=启用`，写审计和 `SupplierUserEnabled`。 |
| 事件 | `SupplierUserEnabled`。 |
| 异常 | 供应商已停用不可启用绑定，返回 `422`。 |

### 4.6 停用绑定

`POST /api/supplier/v1/user-bindings/{supplierUserId}/disable`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `SupplierUserBindingController.disable(id, request)`。 |
| 应用层 | `SupplierUserBindingApplicationService.disable(command)` 校验权限、版本、停用原因。 |
| 领域层 | `SupplierAggregate.disableUserBinding(...)` 校验主账号停用后仍有可用管理员。 |
| 基础设施层 | 更新绑定状态，写审计和事件，通知 IAM 刷新 token/权限缓存。 |
| 事件 | `SupplierUserDisabled`。 |
| 异常 | 无可用管理员返回 `422`。 |

## 5. 供应商商品接口

`GET /api/supplier/v1/items`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `SupplierItemController.page(SupplierItemPageQuery)`。 |

| 领域层 | 无聚合行为。 |

## 继续上下文

当前结论：供应商系统每个接口都应落到明确的 Controller、ApplicationService、Domain、Infrastructure 组件上。查询接口只走 QueryService 和读模型；写接口必须经过应用服务、聚合和本地事务；事件生产使用 Outbox；事件消费使用 Inbox。

关键假设：接口契约以 `docs/06-子系统接口设计/01-供应商系统接口设计.md` 为准；本文负责实现逻辑，不重复所有请求/响应字段。
