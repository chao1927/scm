> 本文承接 [02-采购系统接口事件实现逻辑](02-采购系统接口事件实现逻辑.md)，继续把 `docs/06-子系统接口设计/02-采购系统接口设计.md` 中的接口拆成可编码实现小节。每个接口按“接口层 -> 应用层 -> 领域层 -> 基础设施层 -> 事件/返回 -> 异常”说明，格式对齐供应商系统 `03-供应商系统接口逐项实现设计.md`。

### 1.2 命名约定

| 类型 | 示例 | 说明 |
| --- | --- | --- |
| Controller | `PurchaseOrderController` | 只做协议转换、基础参数校验 |
| QueryService | `PurchaseOrderQueryService` | 只读查询模型，不修改领域状态 |
| ApplicationService | `PurchaseOrderApplicationService` | 编排权限、幂等、事务、聚合、事件、审计 |
| Aggregate | `PurchaseOrderAggregate` | 保护采购订单状态机和数量金额不变量 |
| Repository | `PurchaseOrderRepository` | 保存/加载聚合 |
| DomainEvent | `PurchaseOrderPublishedEvent` | 写入 `pur_domain_event` |
| EventHandler | `WmsInboundEventHandler` | 外部事件消费处理器 |

## 2. 工作台接口

### 2.1 查询工作台统计

`GET /api/purchase/v1/workbench/summary`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `PurchaseWorkbenchController.summary(WorkbenchSummaryQueryRequest)` 解析采购组织、采购组、时间范围和当前用户。 |
| 应用层 | `PurchaseWorkbenchQueryService.querySummary(query)` 校验 `purchase:workbench:read`，追加采购组织、采购组、本人/部门数据范围。 |
| 领域层 | 无聚合行为；这是读模型查询，不触发领域事件。 |
| 基础设施层 | `PurchaseWorkbenchQueryMapper` 聚合查询请购、询价、报价、PO、到货、退供读模型。 |
| 返回 | 待审批请购、待发布询价、待定标比价、待发布 PO、供应商差异、到货异常、退供待办数量。 |
| 异常 | 无权限返回 `403`；数据范围为空返回空统计；查询超时返回 `500` 并记录慢查询。 |

### 2.2 查询待办列表

`GET /api/purchase/v1/workbench/todos`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `PurchaseWorkbenchController.todos(TodoPageQueryRequest)` 接收 `todoType/pageNo/pageSize`。 |
| 应用层 | `PurchaseTodoQueryService.pageTodos(query)` 校验工作台读权限和采购数据范围。 |
| 领域层 | 无聚合行为。 |
| 基础设施层 | 从请购审批、询价截标、比价定标、PO 审批/发布、供应商差异、到货异常、退供审批读模型 union 或统一待办表查询。 |
| 返回 | `todoId`、`businessType`、`businessNo`、`title`、`statusName`、`priority`、`targetRoute`。 |
| 异常 | `pageSize` 超限制返回 `400`；数据范围为空返回空分页。 |

## 3. 请购管理接口

### 3.1 查询请购列表

`GET /api/purchase/v1/requisitions`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `PurchaseRequisitionController.page(RequisitionPageQuery)` 接收状态、申请部门、SKU、时间和分页。 |
| 应用层 | `PurchaseRequisitionQueryService.page(query)` 校验 `purchase:requisition:read`，追加申请人、部门、采购组织范围。 |
| 领域层 | 无聚合行为。 |
| 基础设施层 | 查询 `pur_requisition`、`pur_requisition_line` 读模型，聚合审批摘要和转采购进度。 |
| 返回 | 请购单号、申请组织、状态、审批状态、需求日期、行数、转采购进度。 |
| 异常 | 查询时间跨度过大返回 `400`；无数据权限返回空分页。 |

### 3.2 查询请购详情

`GET /api/purchase/v1/requisitions/{requisitionNo}`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `PurchaseRequisitionController.detail(requisitionNo)` 解析请购单号。 |
| 应用层 | `PurchaseRequisitionQueryService.getDetail(requisitionNo)` 校验读权限和单据组织范围。 |
| 领域层 | 无聚合行为。 |
| 基础设施层 | 查询请购头、行、审批时间线、转采购引用和操作日志摘要。 |
| 返回 | 请购详情、明细、审批记录、关联 RFQ/PO。 |
| 异常 | 单据不存在或无权限返回 `404`。 |

### 3.3 创建请购单

`POST /api/purchase/v1/requisitions`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `PurchaseRequisitionController.create(CreateRequisitionRequest)` 校验请求体和 `X-Idempotency-Key`。 |
| 应用层 | `PurchaseRequisitionApplicationService.create(command)` 校验 `purchase:requisition:create`、幂等、SKU/组织/需求日期。 |
| 领域层 | `PurchaseRequisitionAggregate.create(...)` 校验明细数量大于 0、需求组织有效、同一行 SKU/需求日期可合并规则。 |
| 基础设施层 | `PurchaseRequisitionRepository` 保存头行；写 `pur_domain_event` 和审计。 |
| 事件 | `PurchaseRequisitionCreated`。 |
| 异常 | SKU 停用返回 `422`；幂等键请求内容不同返回 `409 IDEMPOTENCY_CONFLICT`。 |

### 3.4 修改请购单

`PUT /api/purchase/v1/requisitions/{requisitionNo}`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `PurchaseRequisitionController.update(requisitionNo, UpdateRequisitionRequest)` 接收版本号和修改明细。 |
| 应用层 | `PurchaseRequisitionApplicationService.update(command)` 校验修改权限、幂等、版本。 |
| 领域层 | `PurchaseRequisitionAggregate.changeDraft(...)` 只允许草稿/已驳回状态修改，已转采购行不可改。 |
| 基础设施层 | 保存请购头行新版本，写操作审计。 |
| 事件 | 第一版不发布跨系统事件；只写审计。 |
| 异常 | 状态冲突返回 `409 STATE_CONFLICT`；版本冲突返回 `409 VERSION_CONFLICT`。 |

### 3.5 提交请购单

`POST /api/purchase/v1/requisitions/{requisitionNo}/submit`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `PurchaseRequisitionController.submit(requisitionNo, SubmitRequest)` 接收版本和提交说明。 |
| 应用层 | `PurchaseRequisitionApplicationService.submit(command)` 校验提交权限、审批策略和幂等键。 |
| 领域层 | `PurchaseRequisitionAggregate.submit(...)` 校验明细完整，状态草稿/已驳回 -> 待审批。 |
| 基础设施层 | 更新请购状态，写 `pur_domain_event`、审批待办和审计。 |
| 事件 | `PurchaseRequisitionSubmitted`。 |
| 异常 | 明细为空返回 `422`；审批策略缺失返回 `422 APPROVAL_POLICY_MISSING`。 |

### 3.6 审批请购单

`POST /api/purchase/v1/requisitions/{requisitionNo}/approve`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `PurchaseRequisitionController.approve(requisitionNo, ApproveRequest)` 接收审批结论、批准数量和意见。 |
| 应用层 | `PurchaseRequisitionApplicationService.approve(command)` 校验当前审批人、金额/品类审批权限和幂等。 |
| 领域层 | `PurchaseRequisitionAggregate.approve(...)` 推进审批状态，锁定批准数量。 |
| 基础设施层 | 更新请购审批状态，写事件和审计。 |
| 事件 | `PurchaseRequisitionApproved`；驳回场景写 `PurchaseRequisitionRejected`。 |
| 异常 | 非当前审批人返回 `403`；审批已结束返回 `409`。 |

### 3.7 转采购

`POST /api/purchase/v1/requisitions/{requisitionNo}/convert`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `PurchaseRequisitionController.convert(requisitionNo, ConvertRequest)` 接收转 RFQ 或 PO 的目标类型。 |
| 应用层 | `PurchaseRequisitionApplicationService.convert(command)` 校验请购已批准、剩余可转数量和采购权限。 |
| 领域层 | `PurchaseRequisitionAggregate.convert(...)` 记录转采购结果，保证转采购数量不超过批准数量。 |
| 基础设施层 | 创建 RFQ 或 PO 草稿引用，更新请购转采购状态，写事件和审计。 |
| 事件 | `PurchaseRequisitionConverted`。 |
| 异常 | 已完全转采购返回 `409`；目标供应商或 SKU 不可用返回 `422`。 |

## 4. 询价、报价、比价接口

### 4.1 查询询价单列表

`GET /api/purchase/v1/rfqs`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `RfqController.page(RfqPageQuery)` 接收状态、品类、供应商、截止时间和分页。 |
| 应用层 | `RfqQueryService.page(query)` 校验 `purchase:rfq:read` 和采购组织/品类范围。 |
| 领域层 | 无聚合行为。 |
| 基础设施层 | 查询 RFQ 读模型，聚合报价数量、截标状态、定标进度。 |
| 返回 | RFQ 单号、状态、邀请供应商数、已报价数、截止时间、定标状态。 |
| 异常 | 查询跨度过大返回 `400`。 |

### 4.2 创建询价单

`POST /api/purchase/v1/rfqs`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `RfqController.create(CreateRfqRequest)` 校验询价行、邀请供应商和截止时间。 |
| 应用层 | `RfqApplicationService.create(command)` 校验创建权限、幂等、供应商和 SKU 快照。 |
| 领域层 | `RfqAggregate.create(...)` 校验报价截止时间晚于当前时间，邀请供应商不重复。 |
| 基础设施层 | 保存 RFQ 头行、邀请关系，写事件和审计。 |
| 事件 | `RfqCreated`。 |
| 异常 | 供应商停用返回 `422`；截止时间非法返回 `400`。 |

### 4.3 发布询价单

`POST /api/purchase/v1/rfqs/{rfqNo}/publish`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `RfqController.publish(rfqNo, PublishRequest)` 接收版本号和幂等键。 |
| 应用层 | `RfqApplicationService.publish(command)` 校验发布权限、幂等和供应商协同配置。 |
| 领域层 | `RfqAggregate.publish(...)` 状态草稿 -> 已发布/报价中，锁定询价行和邀请供应商。 |
| 基础设施层 | 更新 RFQ 状态，写 `pur_domain_event` 和审计；Outbox 通知供应商系统。 |
| 事件 | `RfqPublished`。 |
| 异常 | 未邀请供应商返回 `422`；重复发布幂等返回历史结果。 |

### 4.4 截标询价单

`POST /api/purchase/v1/rfqs/{rfqNo}/close`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `RfqController.close(rfqNo, CloseRfqRequest)` 接收截标原因。 |
| 应用层 | `RfqApplicationService.close(command)` 校验截标权限、幂等和版本。 |
| 领域层 | `RfqAggregate.closeBidding(...)` 停止接收普通报价，状态 -> 已截标。 |
| 基础设施层 | 更新 RFQ 状态，关闭供应商报价待办，写事件和审计。 |
| 事件 | `RfqBiddingClosed`。 |
| 异常 | 已定标或已取消返回 `409 STATE_CONFLICT`。 |

### 4.5 查询报价列表

`GET /api/purchase/v1/quotes`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `SupplierQuoteController.page(QuotePageQuery)` 接收 RFQ、供应商、状态、分页。 |
| 应用层 | `SupplierQuoteQueryService.page(query)` 校验报价读权限和采购组织范围。 |
| 领域层 | 无聚合行为。 |
| 基础设施层 | 查询 `pur_supplier_quote` 和报价行读模型，补充供应商/SKU 快照。 |
| 返回 | 报价单号、供应商、币种、税率、有效期、报价状态、行数。 |
| 异常 | 供应商快照缺失时返回本地快照并标记 `supplierFresh=false`。 |

### 4.6 提交报价

`POST /api/purchase/v1/quotes/{quoteNo}/submit`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `SupplierQuoteController.submit(quoteNo, SubmitQuoteRequest)` 接收报价版本和幂等键。 |
| 应用层 | `SupplierQuoteApplicationService.submit(command)` 校验报价来源、询价状态、币种税率和幂等。 |
| 领域层 | `SupplierQuotationAggregate.submit(...)` 校验报价行完整、价格大于等于 0、有效期合法。 |
| 基础设施层 | 保存报价状态和行快照，写事件和审计。 |
| 事件 | `SupplierQuotationSubmitted`。 |
| 异常 | RFQ 已截标返回 `409`；价格精度非法返回 `400`。 |

### 4.7 确认报价

`POST /api/purchase/v1/quotes/{quoteNo}/confirm`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `SupplierQuoteController.confirm(quoteNo, ConfirmQuoteRequest)` 接收确认意见。 |
| 应用层 | `SupplierQuoteApplicationService.confirm(command)` 校验采购确认权限和幂等。 |
| 领域层 | `SupplierQuotationAggregate.confirm(...)` 状态已提交 -> 已确认，进入可比价池。 |
| 基础设施层 | 更新报价状态，写事件和审计。 |
| 事件 | `SupplierQuotationConfirmed`。 |
| 异常 | 报价未提交返回 `409 STATE_CONFLICT`。 |

### 4.8 生成比价

`POST /api/purchase/v1/compare-results/generate`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `CompareResultController.generate(GenerateCompareRequest)` 接收 RFQ、比价策略和幂等键。 |
| 应用层 | `CompareResultApplicationService.generate(command)` 校验定标权限、候选报价和策略。 |
| 领域层 | `CompareResultAggregate.generate(...)` 调用比价领域服务计算价格、交期、质量、历史绩效评分。 |
| 基础设施层 | 保存比价结果和评分明细，写事件和审计。 |
| 事件 | `CompareResultGenerated`。 |
| 异常 | 候选报价不足返回 `422`；币种无法换算返回 `422`。 |

### 4.9 定标供应商

`POST /api/purchase/v1/compare-results/{compareNo}/award`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `CompareResultController.award(compareNo, AwardRequest)` 接收中标供应商、报价行和定标理由。 |
| 应用层 | `CompareResultApplicationService.award(command)` 校验定标权限、幂等和审批要求。 |
| 领域层 | `CompareResultAggregate.award(...)` 校验中标报价在候选池内，状态 -> 已定标。 |
| 基础设施层 | 更新比价结果，必要时生成 PO 草稿引用，写事件和审计。 |
| 事件 | `CompareResultAwarded`。 |
| 异常 | 非候选报价不可定标返回 `422`；重复定标返回 `409`。 |

## 5. 采购订单接口

### 5.1 查询采购订单列表

`GET /api/purchase/v1/purchase-orders`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `PurchaseOrderController.page(PurchaseOrderPageQuery)` 接收供应商、状态、仓库、时间和分页。 |
| 应用层 | `PurchaseOrderQueryService.page(query)` 校验 `purchase:po:read`，追加组织、采购组、供应商范围和价格字段权限。 |
| 领域层 | 无聚合行为。 |
| 基础设施层 | 查询 PO 读模型，聚合供应商确认、ASN、入库、退供摘要。 |
| 返回 | PO 单号、供应商、金额、状态、确认状态、入库进度。 |
| 异常 | 价格字段无权限时脱敏；无权限返回空分页。 |

### 5.2 查询采购订单详情

`GET /api/purchase/v1/purchase-orders/{purchaseOrderNo}`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `PurchaseOrderController.detail(purchaseOrderNo)`。 |
| 应用层 | `PurchaseOrderQueryService.getDetail(no)` 校验订单所属组织和供应商范围。 |
| 领域层 | 无聚合行为。 |
| 基础设施层 | 查询 PO 头行、审批、发布、供应商确认、ASN、收货质检上架和操作日志。 |
| 返回 | 订单详情、明细、状态时间线、上下游引用。 |
| 异常 | 不存在或无权限返回 `404`。 |

### 5.3 创建采购订单

`POST /api/purchase/v1/purchase-orders`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `PurchaseOrderController.create(CreatePurchaseOrderRequest)` 校验请求体和幂等键。 |
| 应用层 | `PurchaseOrderApplicationService.create(command)` 校验供应商、SKU、采购价格、仓库、税率和权限。 |
| 领域层 | `PurchaseOrderAggregate.create(...)` 冻结供应商、SKU、价格、仓库快照，校验数量金额大于 0。 |
| 基础设施层 | 保存 `pur_order`、`pur_order_line`，写事件和审计。 |
| 事件 | `PurchaseOrderCreated`。 |
| 异常 | 供应商不可采购、价格失效、仓库停用返回 `422`。 |

### 5.4 提交采购订单

`POST /api/purchase/v1/purchase-orders/{purchaseOrderNo}/submit`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `PurchaseOrderController.submit(purchaseOrderNo, SubmitRequest)` 接收版本和提交说明。 |
| 应用层 | `PurchaseOrderApplicationService.submit(command)` 校验提交权限、审批策略、幂等。 |
| 领域层 | `PurchaseOrderAggregate.submit(...)` 状态草稿 -> 待审批，锁定订单明细。 |
| 基础设施层 | 更新订单状态，生成审批待办，写事件和审计。 |
| 事件 | `PurchaseOrderSubmitted`。 |
| 异常 | 明细为空或金额超权限返回 `422`。 |

### 5.5 审批采购订单

`POST /api/purchase/v1/purchase-orders/{purchaseOrderNo}/approve`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `PurchaseOrderController.approve(purchaseOrderNo, ApproveRequest)` 接收审批结论。 |
| 应用层 | `PurchaseOrderApplicationService.approve(command)` 校验当前审批人、金额权限和版本。 |
| 领域层 | `PurchaseOrderAggregate.approve(...)` 审批通过则状态 -> 已审批；驳回则回到已驳回。 |
| 基础设施层 | 保存审批结果，写事件和审计。 |
| 事件 | `PurchaseOrderApproved` 或 `PurchaseOrderRejected`。 |
| 异常 | 非当前审批人返回 `403`；审批已结束返回 `409`。 |

### 5.6 发布采购订单

`POST /api/purchase/v1/purchase-orders/{purchaseOrderNo}/publish`

## 继续上下文

当前结论：采购 `03` 已按供应商接口逐项格式展开，覆盖工作台、请购、询价报价比价、PO、变更、确认、到货、退供、价格、日志、枚举和跨系统入口。
