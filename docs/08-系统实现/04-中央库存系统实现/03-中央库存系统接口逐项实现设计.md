# 03-中央库存系统接口逐项实现设计

> 本文承接 [02-中央库存系统接口事件实现逻辑](02-中央库存系统接口事件实现逻辑.md)，继续把 `docs/06-子系统接口设计/04-中央库存系统接口设计.md` 中的接口拆成可编码实现小节。每个接口按“接口层 -> 应用层 -> 领域层 -> 基础设施层 -> 事件/返回 -> 异常”说明。

## 1. 统一代码分层

### 1.1 包结构

```text
inventory-service
  ├── interfaces
  │   ├── web
  │   ├── openapi
  │   ├── mq
  │   └── job
  ├── application
  │   ├── command
  │   ├── query
  │   ├── event
  │   └── dto
  ├── domain
  │   ├── account
  │   ├── reservation
  │   ├── freeze
  │   ├── adjustment
  │   ├── snapshot
  │   ├── reconciliation
  │   └── shared
  └── infrastructure
      ├── persistence
      ├── rpc
      ├── mq
      ├── idempotency
      ├── audit
      └── lock
```

### 1.2 命名约定

| 类型 | 示例 | 说明 |
| --- | --- | --- |
| Controller | `InventoryBalanceController` | 前端 HTTP 接口 |
| OpenApiController | `InventoryReservationOpenApiController` | OMS/WMS 等系统调用入口 |
| QueryService | `InventoryBalanceQueryService` | 只读读模型，不改库存 |
| ApplicationService | `InventoryReservationApplicationService` | 编排权限、幂等、账户锁、事务、聚合、事件 |
| Aggregate | `InventoryAccountAggregate` | 保护库存账户数量不变量 |
| DomainEvent | `StockReservedEvent` | 写入 `inv_domain_event` |

## 2. 工作台接口

### 2.1 查询工作台统计

`GET /api/inventory/v1/workbench/summary`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `InventoryWorkbenchController.summary(WorkbenchSummaryQueryRequest)` 解析仓库、货主、时间范围。 |
| 应用层 | `InventoryWorkbenchQueryService.querySummary(query)` 校验 `inventory:workbench:read`，追加仓库/货主数据范围。 |
| 领域层 | 无聚合行为；这是读模型查询。 |
| 基础设施层 | 聚合查询预占异常、冻结审批、调整审批、对账差异、事件失败读模型。 |
| 返回 | 待处理预占、冻结、调整、对账、失败事件数量。 |
| 异常 | 无权限返回 `403`；数据范围为空返回空统计。 |

### 2.2 查询待办列表

`GET /api/inventory/v1/workbench/todos`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `InventoryWorkbenchController.todos(TodoPageQueryRequest)` 接收 `todoType/pageNo/pageSize`。 |
| 应用层 | `InventoryTodoQueryService.pageTodos(query)` 校验工作台读权限和仓库/货主范围。 |
| 领域层 | 无聚合行为。 |
| 基础设施层 | 从预占异常、冻结审批、调整审批、对账差异、事件失败读模型或统一待办表查询。 |
| 返回 | `todoId`、`businessType`、`businessNo`、`warehouseName`、`statusName`、`targetRoute`。 |
| 异常 | `pageSize` 超限制返回 `400`；数据范围为空返回空分页。 |

## 3. 库存余额和可用库存接口

### 3.1 查询库存余额列表

`GET /api/inventory/v1/balances`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `InventoryBalanceController.page(BalancePageQuery)` 接收货主、仓库、SKU、批次、库存状态。 |
| 应用层 | `InventoryBalanceQueryService.page(query)` 校验 `inventory:balance:read` 和仓库/货主权限。 |
| 领域层 | 无聚合行为；查询不能修改库存。 |
| 基础设施层 | 查询库存账户读模型，补充 SKU、仓库快照。 |
| 返回 | 现货、可用、预占、冻结、在途、不可售等数量口径和快照时间。 |
| 异常 | 跨货主查询返回 `403`；查询结果不作为锁库存承诺。 |

### 3.2 查询库存余额详情

`GET /api/inventory/v1/balances/{stockId}`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `InventoryBalanceController.detail(stockId)`。 |
| 应用层 | `InventoryBalanceQueryService.getDetail(stockId)` 校验库存账户所属仓库/货主权限。 |
| 领域层 | 无聚合行为。 |
| 基础设施层 | 查询账户详情、维度、最近流水、预占、冻结和调整摘要。 |
| 返回 | 账户维度、数量口径、最近流水、来源追踪入口。 |
| 异常 | 不存在或无权限返回 `404`。 |

### 3.3 查询可用库存

`POST /api/inventory/v1/available/query`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `AvailableInventoryController.query(AvailableInventoryQueryRequest)` 接收 SKU、仓库范围、渠道/业务场景。 |
| 应用层 | `AvailableInventoryQueryService.queryAvailable(query)` 校验查询权限和可用口径配置。 |
| 领域层 | 无聚合状态变更；可用计算服务按余额、预占、冻结、安全库存、渠道策略计算。 |
| 基础设施层 | 查询账户读模型和策略缓存，必要时补主数据快照。 |
| 返回 | SKU/仓库维度可用数量、不可用原因、口径版本、计算时间。 |
| 异常 | 策略缺失使用默认口径并返回 `policyFallback=true`。 |

### 3.4 导出库存余额

`POST /api/inventory/v1/balances/export`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `InventoryBalanceController.export(BalanceExportRequest)` 接收查询条件。 |
| 应用层 | `InventoryExportApplicationService.createBalanceExport(command)` 校验导出权限和范围上限。 |
| 领域层 | 无聚合行为。 |
| 基础设施层 | 写导出任务，Job 分页查询余额读模型并生成文件。 |
| 返回 | `exportTaskNo`、任务状态。 |
| 异常 | 导出范围过大返回 `400`；任务失败保留失败原因。 |

## 4. 预占接口

### 4.1 查询预占列表

`GET /api/inventory/v1/reservations`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `InventoryReservationController.page(ReservationPageQuery)` 接收来源系统、来源单、仓库、状态。 |
| 应用层 | `InventoryReservationQueryService.page(query)` 校验预占读权限和数据范围。 |
| 领域层 | 无聚合行为。 |
| 基础设施层 | 查询 `inv_reservation`、预占明细、释放/扣减进度。 |
| 返回 | 预占单号、来源单、预占数量、释放数量、扣减数量、状态。 |
| 异常 | 来源单无权限返回 `404` 或空分页。 |

### 4.2 请求库存预占

`POST /openapi/inventory/v1/reservations`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `InventoryReservationOpenApiController.reserve(ReserveStockRequest)` 校验来源系统、签名、幂等键。 |
| 应用层 | `InventoryReservationApplicationService.reserve(command)` 解析库存维度，校验可用数量并获取账户维度锁。 |
| 领域层 | `InventoryReservationAggregate.create(...)` 创建预占；`InventoryAccountAggregate.reserve(...)` 增加预占、减少可用。 |
| 基础设施层 | 同事务保存账户、预占单、库存流水、事件和审计。 |
| 事件 | `StockReserved`；失败策略下写 `StockReservationFailed`。 |
| 异常 | 可用不足按策略全部失败或部分预占；幂等冲突返回 `409`。 |

### 4.3 释放库存预占

`POST /openapi/inventory/v1/reservations/{reservationNo}/release`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `InventoryReservationOpenApiController.release(reservationNo, ReleaseReservationRequest)` 校验来源系统和幂等键。 |
| 应用层 | `InventoryReservationApplicationService.release(command)` 校验来源一致、剩余可释放数量和账户锁。 |
| 领域层 | `InventoryReservationAggregate.release(...)` 释放剩余预占；账户聚合减少预占、增加可用。 |
| 基础设施层 | 保存预占状态、账户、释放流水、事件和审计。 |
| 事件 | `StockReservationReleased`。 |
| 异常 | 已扣减部分只释放剩余；来源不匹配返回 `403`。 |

### 4.4 人工关闭预占

`POST /api/inventory/v1/reservations/{reservationNo}/close`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `InventoryReservationController.close(reservationNo, CloseReservationRequest)` 接收关闭原因。 |
| 应用层 | `InventoryReservationApplicationService.close(command)` 校验异常处理权限和幂等。 |
| 领域层 | `InventoryReservationAggregate.close(...)` 只允许异常或超时预占关闭，并释放剩余预占。 |
| 基础设施层 | 写关闭记录、释放流水、事件和审计。 |
| 事件 | `StockReservationClosed`。 |
| 异常 | 正常履约中的预占不可人工关闭，返回 `422`。 |

## 5. 冻结、调整、流水接口

### 5.1 创建冻结单

`POST /api/inventory/v1/freezes`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `StockFreezeController.create(CreateFreezeRequest)` 校验冻结原因、库存维度、数量和幂等键。 |
| 应用层 | `StockFreezeApplicationService.create(command)` 校验冻结权限、仓库/货主范围和可冻结数量。 |
| 领域层 | `StockFreezeAggregate.create(...)` 创建冻结草稿，锁定冻结明细。 |
| 基础设施层 | 保存冻结单头行，写审计。 |
| 事件 | `StockFreezeCreated`。 |
| 异常 | 数量为 0 或维度非法返回 `422`。 |

### 5.2 审批并执行冻结

`POST /api/inventory/v1/freezes/{freezeNo}/approve`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `StockFreezeController.approve(freezeNo, ApproveFreezeRequest)` 接收审批结论。 |
| 应用层 | `StockFreezeApplicationService.approve(command)` 校验审批权限、幂等和账户锁。 |
| 领域层 | `StockFreezeAggregate.approveAndFreeze(...)` 状态 -> 已冻结；账户聚合增加冻结量、减少可用。 |
| 基础设施层 | 更新冻结单、账户、库存流水，写事件和审计。 |
| 事件 | `StockFrozen`。 |
| 异常 | 可冻结数量不足返回 `422`；版本冲突返回 `409`。 |

### 5.3 解冻库存

`POST /api/inventory/v1/freezes/{freezeNo}/unfreeze`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `StockFreezeController.unfreeze(freezeNo, UnfreezeRequest)` 接收解冻数量和原因。 |
| 应用层 | `StockFreezeApplicationService.unfreeze(command)` 校验解冻权限、幂等和账户锁。 |
| 领域层 | `StockFreezeAggregate.unfreeze(...)` 校验解冻数量不超过冻结剩余。 |
| 基础设施层 | 更新冻结单、账户、解冻流水，写事件和审计。 |
| 事件 | `StockUnfrozen`。 |
| 异常 | 解冻数量超出剩余返回 `422`。 |

### 5.4 执行库存调整

`POST /api/inventory/v1/adjustments/{adjustmentNo}/execute`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `InventoryAdjustmentController.execute(adjustmentNo, ExecuteAdjustmentRequest)` 接收版本和幂等键。 |
| 应用层 | `InventoryAdjustmentApplicationService.execute(command)` 校验已审批、账户锁、负库存策略。 |
| 领域层 | `InventoryAdjustmentAggregate.execute(...)`；账户聚合按调整方向增加或减少库存。 |
| 基础设施层 | 保存调整状态、账户、库存流水、事件和审计。 |
| 事件 | `StockAdjusted`。 |
| 异常 | 负向调整导致负库存返回 `422 NEGATIVE_STOCK_NOT_ALLOWED`。 |

### 5.5 查询库存流水

`GET /api/inventory/v1/ledgers`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `InventoryLedgerController.page(LedgerPageQuery)` 接收来源系统、来源单、账户维度和时间。 |
| 应用层 | `InventoryLedgerQueryService.page(query)` 校验流水读权限和数据范围。 |
| 领域层 | 无聚合行为；流水不可变。 |
| 基础设施层 | 查询库存流水表和事件链路。 |
| 返回 | 流水号、变动方向、变动数量、变动前后数量、来源单、eventId。 |
| 异常 | 时间跨度过大返回 `400`。 |

## 6. 快照和对账接口

### 6.1 生成库存快照

`POST /api/inventory/v1/snapshots`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `InventorySnapshotController.create(CreateSnapshotRequest)` 接收仓库、货主、快照日期。 |
| 应用层 | `InventorySnapshotApplicationService.create(command)` 校验快照权限、幂等和生成中状态。 |
| 领域层 | `InventorySnapshotAggregate.create(...)` 生成快照批次和版本。 |
| 基础设施层 | 写快照头行和明细，写事件和审计。 |
| 事件 | `InventorySnapshotGenerated`。 |
| 异常 | 同日期快照已生成则幂等返回；生成中返回 `409`。 |

### 6.2 重算库存快照

`POST /api/inventory/v1/snapshots/{snapshotNo}/rebuild`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `InventorySnapshotController.rebuild(snapshotNo, RebuildSnapshotRequest)` 接收重算原因。 |
| 应用层 | `InventorySnapshotApplicationService.rebuild(command)` 校验管理员权限和快照使用状态。 |
| 领域层 | `InventorySnapshotAggregate.rebuild(...)` 生成新版本，不覆盖历史版本。 |
| 基础设施层 | 按余额或流水重算，写重算版本和审计。 |
| 事件 | `InventorySnapshotRebuilt`。 |
| 异常 | 已用于对账的快照只能生成新版本，不能覆盖。 |

### 6.3 生成库存对账单

`POST /api/inventory/v1/reconciliations`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `InventoryReconciliationController.create(CreateReconciliationRequest)` 接收对账范围。 |
| 应用层 | `InventoryReconciliationApplicationService.create(command)` 校验对账权限，拉取 WMS 库内库存快照。 |
| 领域层 | `InventoryReconciliationAggregate.create(...)` 比对中央库存和 WMS 快照，生成差异。 |
| 基础设施层 | 保存对账单和差异明细，写事件和审计。 |
| 事件 | `InventoryReconciliationGenerated`。 |
| 异常 | WMS 不可用时生成失败任务，不写半成品差异。 |

### 6.4 确认库存对账

`POST /api/inventory/v1/reconciliations/{reconcileNo}/confirm`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `InventoryReconciliationController.confirm(reconcileNo, ConfirmReconciliationRequest)` 接收确认结论。 |
| 应用层 | `InventoryReconciliationApplicationService.confirm(command)` 校验对账权限、差异处理状态和幂等。 |
| 领域层 | `InventoryReconciliationAggregate.confirm(...)` 锁定差异原因和责任方。 |
| 基础设施层 | 更新对账状态，写事件和审计。 |
| 事件 | `InventoryReconciliationConfirmed`。 |
| 异常 | 未处理差异不可确认。 |

## 7. WMS 记账 OpenAPI

| 事件 | `StockInboundConfirmed`。 |

### 7.2 WMS 出库扣减

`POST /openapi/inventory/v1/outbounds/confirm`

| 层 | 实现逻辑 |
| --- | --- |

| 领域层 | 账户聚合扣减现货和预占；预占聚合记录扣减进度。 |

| 事件 | `StockOutboundConfirmed`。 |

### 7.3 WMS 盘点差异确认

`POST /openapi/inventory/v1/stocktakes/confirm-difference`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `InventoryAccountingOpenApiController.confirmStocktakeDifference(StocktakeDifferenceRequest)` 校验 WMS 来源和幂等键。 |
| 应用层 | `StockAccountingApplicationService.confirmStocktakeDifference(command)` 判断是否需要审批或生成调整单。 |
| 领域层 | 调整聚合或账户聚合处理盘盈盘亏差异。 |
| 基础设施层 | 保存调整/账户/流水/事件和审计。 |
| 事件 | `StocktakeDifferenceConfirmed`、必要时 `StockAdjusted`。 |
| 异常 | 差异超阈值无审批时返回 `403`，不直接改账户。 |

## 8. 事件日志、参数、日志和枚举接口

### 8.1 重放事件

`POST /api/inventory/v1/event-logs/{eventId}/replay`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `InventoryEventLogController.replay(eventId, ReplayEventRequest)` 接收重放原因。 |
| 应用层 | `InventoryEventLogApplicationService.replay(command)` 校验事件管理权限和事件状态。 |

| 基础设施层 | 查询枚举配置表和缓存。 |
| 返回 | 枚举编码、名称、排序、启停状态。 |
| 异常 | 枚举类型不存在返回空列表。 |

## 9. 事件消费接口与监听器

### 9.1 通用事件入口

`POST /internal/inventory/v1/events`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `InventoryEventController.receive(EventEnvelope)` 校验事件信封、来源和版本。 |
| 应用层 | `InventoryEventDispatchService.dispatch(event)` 先写 `inv_event_consume_log`，再分发到 WMS/OMS/主数据事件处理器。 |

| 返回 | `eventId`、`consumeStatus`。 |
| 异常 | 重复事件返回历史结果；缺前置数据进入待重试。 |
