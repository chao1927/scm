# 03-主数据系统接口逐项实现设计

> 本文承接 [02-主数据系统接口事件实现逻辑](02-主数据系统接口事件实现逻辑.md)，继续把 `docs/06-子系统接口设计/08-主数据系统接口设计.md` 中的接口拆成可编码实现小节。每个接口按“接口层 -> 应用层 -> 领域层 -> 基础设施层 -> 事件/返回 -> 异常”说明。

### 1.2 命名约定

| 类型 | 示例 | 说明 |
| --- | --- | --- |
| Controller | `MasterDataRecordController` | 前端 HTTP 接口 |
| OpenApiController | `MdmMasterDataOpenApiController` | 业务系统调用主数据入口 |
| QueryService | `MasterDataRecordQueryService` | 查询读模型，不修改领域状态 |
| ApplicationService | `MasterDataRecordApplicationService` | 编排权限、幂等、模板校验、事务、事件 |
| Aggregate | `MasterDataRecordAggregate` | 保护主数据状态机和关键字段规则 |
| DomainEvent | `MasterDataEnabledEvent` | 写入 `mdm_domain_event` |
| EventHandler | `ApprovalApprovedHandler` | 外部事件消费处理器 |

## 2. 类型、字段模板和编码规则接口

### 2.1 查询主数据类型列表和详情

`GET /api/mdm/v1/types`、`GET /api/mdm/v1/types/{typeCode}`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `MasterDataTypeController.page/detail` 接收类型编码、领域、状态、分页或路径类型编码。 |
| 应用层 | `MasterDataTypeQueryService.page/detail(query)` 校验 `mdm:type:read` 和系统配置范围。 |
| 领域层 | 无聚合行为。 |
| 基础设施层 | 查询 `mdm_master_data_type`，详情补充字段模板数量、启用记录数、状态时间线。 |
| 返回 | 类型编码、名称、领域、审批、版本、发布、状态、排序、状态时间线、版本号。 |
| 异常 | 数据范围为空返回空分页；详情不存在返回 `404`；非法领域枚举返回 `400`。 |

### 2.2 新增或修改主数据类型

`POST /api/mdm/v1/types`、`PUT /api/mdm/v1/types/{typeCode}`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `MasterDataTypeController.create/update` 校验请求体和 `X-Idempotency-Key`。 |
| 应用层 | `MasterDataTypeApplicationService.save(command)` 校验按钮权限、幂等、类型编码唯一和乐观锁。 |
| 领域层 | `MasterDataTypeAggregate.create/change(...)` 校验领域、上级类型、发布开关和状态机。 |
| 基础设施层 | 保存 `mdm_master_data_type`，写 `mdm_domain_event` 和 `mdm_operation_audit_log`。 |
| 事件 | `MasterDataTypeCreated`、`MasterDataTypeChanged`。 |
| 异常 | 类型编码重复返回 `409`；启用后不允许修改关键配置时返回 `422`。 |

### 2.3 启用或停用主数据类型

`POST /api/mdm/v1/types/{typeCode}/enable`、`POST /api/mdm/v1/types/{typeCode}/disable`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | 接收类型编码、原因和版本。 |
| 应用层 | 校验 `master-data:master_data:enable/disable`、幂等和乐观锁。 |
| 领域层 | `MasterDataTypeAggregate.enable/disable` 校验停用前无启用字段模板和待发布记录，或要求审批。 |
| 基础设施层 | 更新状态，写事件和审计。 |
| 事件 | `MasterDataTypeEnabled`、`MasterDataTypeDisabled`。 |
| 异常 | 被启用记录引用且不允许停用时返回 `422 BUSINESS_RULE_FAILED`。 |

### 2.4 字段模板查询、新增、修改、排序和停用

`GET /api/mdm/v1/field-defs`、`GET /api/mdm/v1/field-defs/{fieldDefinitionId}`、`POST /api/mdm/v1/field-defs`、`PUT /api/mdm/v1/field-defs/{fieldDefinitionId}`、`POST /api/mdm/v1/field-defs/sort`、`POST /api/mdm/v1/field-defs/{fieldDefinitionId}/disable`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `FieldDefinitionController` 接收查询条件、字段编码、类型、必填、唯一、枚举、引用和排序。 |
| 应用层 | 查询走 `FieldDefinitionQueryService`；写命令走 `FieldDefinitionApplicationService`，校验类型已启用、字段编码类型内唯一、字段类型和引用规则。 |
| 领域层 | 查询无聚合行为；写命令由 `FieldDefinitionAggregate` 校验启用后字段可改性、关键字段规则和表单展示约束。 |
| 基础设施层 | 保存 `mdm_field_definition`，刷新字段模板缓存，写事件和审计。 |
| 返回/事件 | 查询返回字段模板详情、枚举/引用配置和可变性；写命令产生 `FieldTemplateCreated`、`FieldTemplateChanged`、`FieldTemplateDisabled`、`EnumItemChanged`。 |
| 异常 | 字段不存在返回 `404`；字段已被启用主数据使用且不可改时返回 `422`；排序版本冲突返回 `409`。 |

### 2.5 编码规则查询、维护和编码生成

`GET /api/mdm/v1/code-rules`、`GET /api/mdm/v1/code-rules/{ruleCode}`、`POST /api/mdm/v1/code-rules`、`PUT /api/mdm/v1/code-rules/{ruleCode}`、`POST /api/mdm/v1/code-rules/{ruleCode}/enable`、`POST /api/mdm/v1/code-rules/{ruleCode}/disable`、`POST /api/mdm/v1/code-rules/{ruleCode}/preview`、`POST /openapi/mdm/v1/codes/generate`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `CodeRuleController` 和 `MdmCodeOpenApiController` 接收查询条件、规则、启停原因、业务日期、变量和生成数量。 |
| 应用层 | 查询走 `CodeRuleQueryService`；写命令校验规则权限、幂等、启用状态和乐观锁；生成接口按 `ruleCode + bizKey` 幂等。 |
| 领域层 | `CodeRuleAggregate.enable/disable/generate(...)` 校验状态机，按前缀、日期段、流水长度、重置周期生成编码；预览不占用流水。 |
| 基础设施层 | 使用数据库行锁或 Redis 分布式锁更新 `current_sequence`，保存事件和审计。 |
| 事件 | `CodeRuleCreated`、`CodeRuleEnabled`、`MasterDataCodeGenerated`、`CodeRuleDisabled`。 |
| 异常 | 流水并发冲突重试；规则停用返回 `422`；同幂等键参数不同返回 `409`。 |

## 3. 主数据记录、审核和版本接口

### 3.1 查询主数据列表和详情

`GET /api/mdm/v1/records`、`GET /api/mdm/v1/records/{recordId}`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `MasterDataRecordController.page/detail` 接收领域、类型、编码、名称、状态、审批状态。 |
| 应用层 | `MasterDataRecordQueryService` 校验商品/伙伴/仓储页面读权限和数据范围。 |
| 领域层 | 无聚合行为。 |
| 基础设施层 | 查询 `mdm_master_data_record`、字段模板、版本快照和状态时间线。 |
| 返回 | 主数据编码、名称、业务字段、状态、审批状态、当前版本、生效/失效时间。 |
| 异常 | 无数据权限按 `404` 或空分页处理，避免泄露存在性。 |

### 3.2 新增或修改主数据

`POST /api/mdm/v1/records`、`PUT /api/mdm/v1/records/{recordId}`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `MasterDataRecordController.create/update` 接收 `typeCode/dataCode/dataPayload` 和附件。 |
| 应用层 | `MasterDataRecordApplicationService.save(command)` 校验权限、幂等、字段模板、唯一字段、引用主数据、乐观锁。 |
| 领域层 | `MasterDataRecordAggregate.create/change(...)` 判断普通字段立即保存，关键字段生成变更记录等待审批。 |
| 基础设施层 | 保存记录、变更日志、审计；必要时生成编码。 |
| 事件 | `MasterDataDraftCreated`、`MasterDataChangeSubmitted`、`DataQualityIssueFound`。 |
| 异常 | 引用主数据未启用返回 `422 REFERENCE_NOT_ENABLED`；编码重复返回 `409`。 |

### 3.3 审核列表、提交审核、审核通过和驳回

`GET /api/mdm/v1/approvals`、`GET /api/mdm/v1/approvals/{changeLogId}`、`POST /api/mdm/v1/records/{recordId}/submit`、`POST /api/mdm/v1/approvals/{changeLogId}/approve`、`POST /api/mdm/v1/approvals/{changeLogId}/reject`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | 审核查询接收类型、编码、字段、申请人和时间；提交审核接收说明；审核接口接收审批意见、驳回原因和版本。 |
| 应用层 | 查询走 `MasterDataApprovalQueryService`；提交时调用权限/审批 OpenAPI 创建审批任务；审批回调或页面审批统一进入 `MasterDataApprovalApplicationService`。 |
| 领域层 | 查询无聚合行为；写命令由 `MasterDataRecordAggregate.submit/approve/reject` 校验状态机：草稿 -> 待审核 -> 已启用/已驳回。 |
| 基础设施层 | 更新记录、变更日志，审核通过生成 `mdm_master_data_version` 和 `mdm_publish_log`。 |
| 返回/事件 | 查询返回变更前后快照、关键字段标识和审批状态；写命令产生 `MasterDataSubmitted`、`MasterDataEnabled`、`MasterDataRejected`、`MasterDataVersionGenerated`。 |
| 异常 | 已启用记录重复审核返回 `409`；审批系统超时保留待提交补偿任务。 |

### 3.4 启用、冻结和停用主数据

`POST /api/mdm/v1/records/{recordId}/enable`、`POST /api/mdm/v1/records/{recordId}/freeze`、`POST /api/mdm/v1/records/{recordId}/disable`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | 接收生效时间、冻结/停用原因和版本。 |
| 应用层 | 校验按钮权限、数据范围、是否存在下游引用、幂等和乐观锁。 |
| 领域层 | 聚合推进状态，停用前检查引用关系和是否允许补偿审批。 |
| 基础设施层 | 更新记录状态，生成版本，按订阅生成发布日志，写审计。 |
| 事件 | `MasterDataEnabled`、`MasterDataFrozen`、`MasterDataDisabled`、派生事件如 `SupplierFrozen`、`SkuDisabled`。 |
| 异常 | 已被业务单据引用且禁止停用时返回 `422`；需要审批时返回 `202 ACCEPTED`。 |

## 4. 发布、导入导出和日志接口

### 4.1 查询发布日志和重试发布

`GET /api/mdm/v1/publish-logs`、`GET /api/mdm/v1/publish-logs/{publishLogId}`、`POST /api/mdm/v1/publish-logs/{publishLogId}/retry`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `MasterDataPublishController` 接收目标系统、发布状态、版本、发布日志ID和重试原因。 |
| 应用层 | 查询校验发布权限；详情读取发布载荷预览、回执和失败原因；重试校验日志状态、幂等和目标订阅仍启用。 |
| 领域层 | 查询无聚合行为；重试由 `PublishSubscriptionAggregate.retry(...)` 推进发布状态为待发布/发布中。 |
| 基础设施层 | 更新 `mdm_publish_log`，写 `MasterDataRepublished` 和审计，发布任务再次投递。 |
| 返回 | 发布日志、失败原因、重试次数、目标系统。 |
| 异常 | 成功状态重复重试返回 `409 STATE_CONFLICT`。 |

### 4.2 发布订阅配置

`GET /api/mdm/v1/publish-subscriptions`、`POST /api/mdm/v1/publish-subscriptions`、`PUT /api/mdm/v1/publish-subscriptions/{subscriptionId}`、`POST /api/mdm/v1/publish-subscriptions/{subscriptionId}/toggle`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | 接收类型、目标系统、主题、发布模式、过滤规则。 |
| 应用层 | 校验系统配置权限、类型存在、目标系统合法、过滤规则 JSON 可执行。 |
| 领域层 | 发布订阅聚合保证同一 `typeCode + targetSystem + eventTopic` 不重复启用。 |
| 基础设施层 | 保存 `mdm_publish_subscribe`，刷新发布订阅缓存，写审计。 |
| 事件 | `SubscriptionCreated`、`SubscriptionDisabled`。 |
| 异常 | 过滤规则非法返回 `400`；重复启用返回 `409`。 |

### 4.3 导入任务、模板、错误明细和主数据导出

`GET /api/mdm/v1/import-tasks`、`GET /api/mdm/v1/import-tasks/{importTaskId}`、`GET /api/mdm/v1/import-templates/{typeCode}`、`POST /api/mdm/v1/import-tasks`、`POST /api/mdm/v1/import-tasks/{importTaskId}/execute`、`POST /api/mdm/v1/import-tasks/{importTaskId}/cancel`、`GET /api/mdm/v1/import-tasks/{importTaskId}/errors`、`POST /api/mdm/v1/records/export`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | 接收导入查询条件、类型、文件地址、文件哈希、导入模式、重复策略、导出过滤条件和字段清单。 |
| 应用层 | 查询走 `ImportTaskQueryService`；模板按字段定义动态生成；导入由 `ImportTaskApplicationService` 校验权限、幂等、文件格式、字段模板，创建任务并校验行；导出走异步导出任务。 |
| 领域层 | 导入任务聚合推进待处理、处理中、成功、部分成功、失败、取消状态；导出不修改主数据聚合。 |
| 基础设施层 | 文件服务读取 Excel/CSV，逐行转换为主数据命令，保存成功行、错误文件和导出文件地址。 |
| 返回/事件 | 返回任务进度、错误明细、模板下载地址或导出任务号；事件为 `ImportTaskCreated`、`ImportFileValidated`、`ImportTaskExecuted`、`ImportTaskCompleted`、`ImportTaskCancelled`。 |
| 异常 | 部分失败保留成功数据和错误文件；错误文件过期重新生成；同文件哈希重复导入按幂等返回历史任务；导出量过大返回 `202`。 |

### 4.4 变更日志、操作日志和枚举

`GET /api/mdm/v1/change-logs`、`GET /api/mdm/v1/change-logs/{changeLogId}`、`POST /api/mdm/v1/change-logs/export`、`GET /api/mdm/v1/operation-logs`、`GET /api/mdm/v1/operation-logs/{operationLogId}`、`POST /api/mdm/v1/operation-logs/export`、`GET /api/mdm/v1/enums`、`POST /api/mdm/v1/enums`、`PUT /api/mdm/v1/enums/{enumItemId}`、`POST /api/mdm/v1/enums/sort`、`POST /api/mdm/v1/enums/{enumItemId}/disable`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | 日志接口接收对象、字段、操作人、时间、详情ID和导出字段；枚举接口接收类型、值、标签、排序和版本。 |
| 应用层 | 日志查询校验审计权限和脱敏策略，导出创建异步任务；枚举写接口校验非核心枚举才允许增删改。 |
| 领域层 | 日志查询无聚合行为；枚举规则服务校验枚举值唯一、核心状态枚举不可删除。 |
| 基础设施层 | 查询变更日志和审计表；枚举写入配置读模型和字段模板缓存。 |
| 返回/事件 | 日志详情返回前后快照和失败原因；枚举写命令产生 `EnumItemChanged`。 |
| 异常 | 核心枚举停用返回 `422`；日志导出数据量过大创建异步导出任务。 |

## 5. OpenAPI 和事件入口

### 5.1 主数据查询、校验和快照接口

`POST /openapi/mdm/v1/master-data/query`、`POST /openapi/mdm/v1/master-data/validate`、`GET /openapi/mdm/v1/master-data/{typeCode}/{dataCode}`、`GET /openapi/mdm/v1/suppliers/{supplierId}`、`GET /openapi/mdm/v1/warehouses/{warehouseId}/locations`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `MdmMasterDataOpenApiController` 校验来源系统、签名、appCode、通用查询参数、供应商ID、仓库ID和库位过滤条件。 |
| 应用层 | `MdmOpenQueryService` 校验 OpenAPI 权限、数据范围和目标类型是否允许开放；供应商/仓库接口映射到 `SUPPLIER/WAREHOUSE/LOCATION` 类型快照。 |
| 领域层 | 无聚合行为。 |
| 基础设施层 | 查询记录当前快照或版本快照，批量校验引用关系；供应商接口补资质/结算摘要，仓库接口分页返回库区库位。 |
| 返回 | 主数据快照、校验结果、失败明细、供应商主档、仓库库位列表。 |
| 异常 | 来源系统未授权返回 `403`；类型未开放返回 `422`。 |

### 5.2 发布回执接口

`POST /openapi/mdm/v1/publish-receipts`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `MdmPublishReceiptOpenApiController.receipt` 校验来源系统、事件ID、版本和目标系统。 |
| 应用层 | `MasterDataPublishApplicationService.receiveReceipt(command)` 幂等处理成功、失败、忽略回执。 |
| 领域层 | 发布订阅聚合更新目标系统发布状态，不改变主数据业务状态。 |
| 基础设施层 | 更新 `mdm_publish_log`，写消费日志、审计和确认事件。 |
| 事件 | 成功回执产生 `MasterDataPublishConfirmed`；失败不产生成功事件但记录失败原因。 |
| 异常 | 版本不匹配返回 `409`；未知发布日志返回 `404`。 |

### 5.3 通用事件入口

`POST /internal/mdm/v1/events`

| 层 | 实现逻辑 |
| --- | --- |
| 接口层 | `MdmEventConsumer.onMessage` 解析事件信封，校验事件类型和来源系统。 |
| 应用层 | `MdmEventDispatchService` 先写 `mdm_event_consume_log`，再分派到审批、权限、发布回执、供应商资料、引用关系处理器。 |
| 领域层 | 消费处理器加载对应聚合推进本地状态或更新引用快照。 |
| 基础设施层 | 保存消费日志、业务状态、审计；失败更新重试次数。 |
| 返回 | `consumeId`、`consumeStatus`、`idempotentHit`。 |
| 异常 | 不支持事件类型返回已忽略；领域失败可重试，格式错误不可重试。 |

## 继续上下文

当前结论：主数据逐项实现已覆盖类型、字段、编码、记录、审核、发布、导入导出、日志、OpenAPI 和事件入口。
