# FE-REQ-006 主数据与权限管理页面

## 1. 背景

MDM 和 IAM 后端已完成首轮接口级能力。当前前端已覆盖供应商 ASN、集成中心、采购、WMS/库存、OMS/TMS/BMS，但尚未提供主数据与权限管理入口。

## 2. 目标

- 打开主数据与权限管理菜单。
- 提供主数据类型、字段模板、编码规则、主数据记录、导入任务、质量问题、发布订阅、发布记录的运营视图。
- 提供用户、角色、权限点、审批实例、安全策略、应用、SSO 客户端的权限运营视图。

## 3. 范围

| 系统 | 范围 |
| --- | --- |
| MDM | 类型、模板、编码规则、记录、导入任务、质量问题、发布订阅、发布记录 |
| IAM | 用户、角色、权限、审批、操作日志、安全策略、应用、SSO 客户端 |

## 4. 不在本轮范围

- 主数据创建/审核/冻结/禁用操作表单。
- 用户、角色、权限、菜单、SSO 的创建与授权操作表单。
- OpenAPI 权限校验模拟器。
- Web 权限自动化测试。

## 5. 验收标准

- “主数据”和“权限管理”菜单可进入同一管理页面。
- 页面能调用 `/api/mdm/v1`、`/api/iam/v1` 的已完成读接口。
- `npm run build`、后端 `mvn test -q`、`git diff --check` 通过。

## 6. 风险

- 本轮仍是运营查询面，不替代完整后台配置工作台。
- IAM 与 MDM 的接口返回包装不完全一致，前端需统一兼容。
- 权限 Web 自动化测试需单独进入后续需求单。
