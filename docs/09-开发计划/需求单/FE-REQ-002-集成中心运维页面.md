# FE-REQ-002 集成中心运维页面

## 1. 背景

集成中心后端已完成路由配置、端点配置、消息接入、投递运行面、尝试日志、死信重放和运维汇总接口。当前前端仅有运营工作台和供应商 ASN 页面，无法让运维人员直接查看集成链路健康度、处理失败消息和执行端点预检。

## 2. 目标

- 建立集成中心前端入口，覆盖日常运维高频动作。
- 将已完成的集成中心后端能力暴露为可操作页面。
- 明确真实 RocketMQ、Dubbo、ERP、税务、支付沙箱仍属于外部环境阻塞，不在本需求中伪造完成。

## 3. 范围

| 范围 | 说明 |
| --- | --- |
| 运维汇总 | 展示待投递、已投递、失败、死信、已重放数量 |
| 路由管理 | 查询路由、新建路由、停用路由 |
| 端点管理 | 查询端点、新建端点、停用端点、端点契约预检 |
| 消息运维 | 查询消息、按状态筛选、手工派发、重试失败消息 |
| 死信处理 | 查询死信、人工重放 |
| 投递运行面 | 执行批量投递，查看扫描、成功、失败、死信数量 |
| 尝试日志 | 查询投递尝试，支持按消息号筛选 |

## 4. 不在本轮范围

- 真实 RocketMQ Producer、Dubbo 泛化调用 SDK 接入。
- ERP、税务、支付沙箱真实联调。
- 前端路由级权限守卫和 E2E Web 权限测试。
- 消息接入 OpenAPI 的外部调用模拟页。

## 5. 验收标准

- 菜单“集成中心”可进入运维页面。
- 页面能调用集成中心 `/api/integration/v1` 下的管理接口。
- 路由、端点、消息、死信、尝试日志均有表格视图和关键操作。
- 创建、停用、预检、派发、重试、重放、批量投递操作成功后刷新相关查询。
- `npm run build`、后端 `mvn test -q`、`git diff --check` 通过。

## 6. 风险

- 后端管理接口当前返回原始列表/对象，不是统一分页包；前端本轮按真实返回结构适配。
- 没有真实后端服务运行时，页面会展示请求失败状态；不影响静态构建验收。
- Ant Design 仍可能产生 Vite chunk size 警告，后续通过路由拆包处理。
