# OMS-REQ-001 渠道订单接入与销售订单审核

## 目标

完成 OMS 第一条可验收闭环：渠道订单幂等接入，生成内部销售订单，并支持订单运营审核通过或拦截。

## 范围

- 渠道订单：按 `channelCode + channelOrderNo` 幂等接入。
- 销售订单：保存客户、收货地址、订单行、金额和状态。
- 审单：支持审核通过、拦截并记录原因。
- 事件与审计：写入 OMS Outbox 和操作日志。

## 业务规则

| 规则 | 说明 |
| --- | --- |
| 渠道订单幂等 | 同一渠道订单重复接入返回既有销售订单 |
| 订单行必填 | 至少一行，SKU、数量、单价有效 |
| 金额不可为负 | 订单总金额由行金额汇总 |
| 待审核才能审单 | 已审核、已拦截、已取消订单不能重复审单 |
| 拦截必须有原因 | 风控、地址、商品、价格异常需记录原因 |

## 接口

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `POST` | `/api/oms/v1/channel-orders` | 接入渠道订单 |
| `GET` | `/api/oms/v1/channel-orders` | 查询渠道订单映射 |
| `GET` | `/api/oms/v1/sales-orders` | 查询销售订单 |
| `GET` | `/api/oms/v1/sales-orders/{orderNo}` | 查询销售订单详情 |
| `POST` | `/api/oms/v1/sales-orders/{orderNo}/review` | 审核通过或拦截 |

## 验收

- 领域测试覆盖渠道单幂等、订单行校验、审核状态机和拦截原因。
- 应用服务测试覆盖接单、重复接单、审核、Outbox 和审计。
- 全量 `mvn test -q` 通过。
