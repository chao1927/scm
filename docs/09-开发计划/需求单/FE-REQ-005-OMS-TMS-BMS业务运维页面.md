# FE-REQ-005 OMS、TMS、BMS 业务运维页面

## 1. 背景

OMS、TMS、BMS 后端均已完成首轮接口级能力，但前端尚未提供订单履约、物流运输和费用结算的统一运营视图。当前前端已完成工作台、供应商 ASN、集成中心、采购中心、WMS 与中央库存运维页面，下一步需要补齐履约后段业务面。

## 2. 目标

- 打开 OMS、TMS、BMS 菜单入口。
- 提供订单履约、物流运输、费用结算的可查运营页面。
- 本轮优先接真实读接口，避免用默认参数误推进业务状态。

## 3. 范围

| 系统 | 范围 |
| --- | --- |
| OMS | 渠道订单、销售订单、履约单、出库单 |
| TMS | 运输任务、运单、物流异常、费用来源 |
| BMS | 计费对象、费用来源、结算汇总 |

## 4. 不在本轮范围

- OMS 审单、履约分仓、出库派发、取消售后的复杂操作表单。
- TMS 接单、生成运单、面单、轨迹、签收、费用推送的操作表单。
- BMS 计费规则、费用重算/作废、对账、账单、开票、财务交接、退款的完整操作页。
- 权限 Web 自动化测试。

## 5. 验收标准

- “订单履约”“物流运输”“费用结算”菜单可进入同一履约结算运营页面。
- 页面能调用 `/api/oms/v1`、`/api/tms/v1`、`/api/bms/v1` 的已完成读接口。
- `npm run build`、后端 `mvn test -q`、`git diff --check` 通过。

## 6. 风险

- 首轮页面偏运营查询，不代替完整业务操作工作台。
- BMS 结算汇总需要时间范围，本轮使用当天所属年度默认区间。
- 后续仍需按系统补操作表单和权限 Web 测试。
