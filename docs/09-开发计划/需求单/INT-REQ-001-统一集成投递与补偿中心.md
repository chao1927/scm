# INT-REQ-001 统一集成投递与补偿中心

## 目标

补齐供应链平台统一集成生产化底座，集中承接跨系统领域事件和同步命令的路由、派发、失败记录、死信和人工重放，避免各业务系统各自实现一套不可观测的投递补偿逻辑。

## 范围

- 集成路由：按 `eventType/commandType + sourceSystem` 配置目标系统和通道。
- 统一消息：事件或命令进入后生成可派发消息，记录幂等键、业务号、载荷和状态。
- 派发状态：支持待派发、已派发、派发失败、死信、已重放。
- 死信补偿：失败超过阈值进入死信，可人工重放生成新的待派发消息。
- 运维接口：查询消息、路由、死信，手工派发、重试、重放。

## 验收

- 同一 `sourceSystem + idempotencyKey` 重复进入返回既有消息。
- 未配置路由的事件进入死信，不丢载荷。
- 派发失败累计到阈值后进入死信。
- 死信重放生成新消息并保留原死信记录。
- 领域、应用、接口测试和全量 Maven 测试通过。
