# FE-REQ-007 前端权限拦截与基础 Web 测试

## 1. 背景

前端核心业务页面已完成首轮覆盖，但菜单仍是静态可进入状态。IAM 后端已具备用户、角色、权限点、权限快照和 Token 校验能力，前端需要先建立基础权限拦截与可测试的权限规则，为后续真实登录态和 Web E2E 权限测试打底。

## 2. 目标

- 将菜单权限规则从 `App.jsx` 中抽离为可测试纯函数。
- 在 sessionStorage 存在权限码时按权限码禁用无权限菜单。
- 没有权限码时保留开发环境全菜单可用，避免当前无登录页场景下阻断开发验收。
- 增加 Vitest 单元测试覆盖权限码解析、通配权限、菜单禁用逻辑。

## 3. 范围

| 范围 | 说明 |
| --- | --- |
| 菜单权限规则 | 为核心菜单定义权限码 |
| 权限码解析 | 支持 JSON 数组和逗号分隔字符串 |
| 菜单装饰 | 根据权限码设置叶子菜单 disabled |
| 基础测试 | 使用 Vitest 覆盖核心规则 |

## 4. 不在本轮范围

- 登录页和真实 Token 获取流程。
- 浏览器 E2E 自动化。
- 后端权限接口联调。
- 动态菜单从 IAM 接口加载。

## 5. 验收标准

- `App.jsx` 使用权限工具装饰菜单。
- `npm run test`、`npm run build`、后端 `mvn test -q`、`git diff --check` 通过。

## 6. 风险

- 本轮只是前端基础权限规则，不代表完整安全边界；最终权限仍以后端鉴权为准。
- 没有权限码时默认全量可用是开发态兼容策略，生产上线前需要接入真实登录和权限快照。
