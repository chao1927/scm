# OMS-REQ-004 外部订单查询与事件入口

## 1. 需求目标

为渠道、客服和运营提供统一的订单履约查询视图，并为中央库存、WMS、TMS、BMS 事实回传提供版本化、幂等的 OMS 内部事件入口。

## 2. 本期范围

- `GET /openapi/orders/{orderNo}` 查询订单、履约、预占和出库摘要。
- `GET /fulfillments/{fulfillmentNo}/tracking` 查询履约当前状态和 WMS 处理状态。
- `POST /internal/oms/v1/events` 统一接收库存、WMS、退款事件，由 Inbox 幂等后路由到对应应用服务。

## 3. 验收标准

- 查询接口不修改写模型，返回订单不存在、履约不存在和部分事实未到达时的明确状态。
- 查询结果包含订单号、履约单号、仓库、履约状态、预占引用、出库单号、WMS 单号和出库状态。
- 事件入口按事件类型路由；重复 `eventId` 不重复推进聚合；未知事件拒绝并记录失败原因。
- OpenAPI 查询和内部事件入口分别使用外部调用范围与内部来源校验，后续接入 IAM/签名 ACL。
