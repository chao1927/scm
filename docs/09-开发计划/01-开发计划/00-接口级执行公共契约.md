# 接口级开发执行公共契约

本文件不替代系统接口小节；它定义所有接口都必须具备的共同骨架。各系统接口级计划只描述该接口的业务差异、具体类、事件和外部协作。

## 1. 写接口共同链路

```mermaid
sequenceDiagram
  participant U as 用户或调用系统
  participant C as 接口层 Controller/OpenAPI
  participant A as 应用服务
  participant D as 聚合或领域服务
  participant R as 资源库和数据库
  participant O as Outbox
  participant M as 消息队列
  U->>C: 请求头、DTO、幂等键
  C->>C: 鉴权、格式和参数校验
  C->>A: Command和权限上下文
  A->>A: 数据范围、幂等、事务
  A->>R: 加载聚合
  A->>D: 执行业务命令和不变量校验
  D-->>A: 新状态和领域事件
  A->>R: 乐观锁保存和审计
  A->>O: 同事务保存事件
  A-->>C: CommandResult
  C-->>U: 统一响应
  O->>M: 异步发布已发生事实
```

## 2. 查询接口共同链路

```mermaid
sequenceDiagram
  participant U as 用户或调用系统
  participant C as 接口层
  participant Q as 查询应用服务
  participant RM as 读模型或缓存
  participant ACL as 可选外部查询ACL
  U->>C: 查询条件和访问令牌
  C->>Q: Query和权限上下文
  Q->>Q: 校验权限、数据范围、分页和排序
  Q->>RM: 查询投影或读表
  opt 需要补充外部快照
    Q->>ACL: 只读RPC调用
  end
  Q-->>C: PageResult或DetailView
  C-->>U: 统一响应
```

## 3. 入站事件共同链路

```mermaid
sequenceDiagram
  participant M as 消息队列或回调方
  participant L as Listener/OpenAPI
  participant I as Inbox
  participant A as 事件应用服务
  participant D as 聚合
  participant R as 数据库
  M->>L: 事件编码、来源、载荷、版本
  L->>I: 声明消费幂等键
  alt 已成功
    I-->>L: 忽略重复事件
  else 可处理
    L->>A: Event
    A->>D: 转换为领域命令
    D->>R: 保存状态/投影
    A->>I: 标记成功
  else 失败
    A->>I: 保存原始载荷和失败原因
    L-->>M: FAILURE或错误响应，进入重试/死信
  end
```

## 4. 强制实现项

- 写接口：`X-Request-Id`、`X-Trace-Id`、`X-Idempotency-Key`、权限点、数据范围、版本号。
- 查询接口：页码、页大小、总数、排序白名单、组织/仓/货主/供应商/客户范围。
- 入站事件：来源校验、事件编码、事件版本、Inbox、原始载荷、失败重放。
- 出站事件：聚合版本、领域事件、Outbox、投递状态、重试与人工重放。
- 代码：Controller、Command/Query DTO、ApplicationService/QueryService、Aggregate/DomainService、Repository/Mapper、ACL、Listener、测试类均需在系统小节中点名。
