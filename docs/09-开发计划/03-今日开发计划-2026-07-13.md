# 2026-07-13 供应链系统开发计划

## 1. 计划目标

根据 2026-07-13 功能盘点结果，将“首轮代码切片完成”与“生产可用完成”分开管理。本计划补入统一鉴权、持久化幂等、调拨、售后逆向、真实集成、集成测试和运营治理等遗漏项，并作为今日起逐项更新状态的执行看板。

状态只使用：`待开发`、`进行中`、`已完成`、`阻塞`。任务只有同时满足验收条件、测试通过并更新开发日志后，才可标记为 `已完成`。

## 2. 领域驱动设计对齐说明

| 项目 | 对齐口径 |
| --- | --- |
| 核心域 | 库存可用性、销售履约、仓内作业、调拨协同、逆向退货 |
| 支撑域 | 采购、供应商协同、运输、计费结算、主数据 |
| 通用域 | 权限、集成、审计、附件、通知、报表与运维治理 |
| 数据主权 | 调拨单由中央库存上下文拥有；售后单由 OMS 拥有；退货仓内事实由 WMS 拥有；库存处置由中央库存拥有 |
| 跨上下文规则 | 请求动作使用命令，状态结果使用领域事件；所有外部命令和事件必须有幂等键、Outbox/Inbox、重试和人工补偿 |
| 完成口径 | 领域模型、应用服务、接口、持久化、权限、事件、测试、文档和日志全部验收通过 |

## 4. 执行规则

1. 严格按表中顺序执行；同一依赖层的独立切片可并行，但共享契约必须先完成。
2. 开始任务时将状态更新为 `进行中`，同时在 `tasks/todo.md` 标记进行中。
3. 完成任务后运行该切片测试和全量回归，更新接口、事件、DDL 差异及 `02-开发日志.md`，再标记为 `已完成`。
4. 外部环境缺失只允许标记 `阻塞`，不得用本地模拟结果替代真实联调验收。
5. 任一 P0 任务未完成前，不宣称端到端供应链系统完成。
