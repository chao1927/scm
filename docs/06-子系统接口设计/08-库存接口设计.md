## 1. DDD 对齐说明

| DDD 关注点 | 本文口径 |
| --- | --- |
| 限界上下文 | 中央库存上下文 |
| 聚合根 | `InventoryAccount`、`InventoryReservation` |
| 实体/值对象 | 库存维度、库存数量、库存状态、来源单据、幂等键、流水批次 |
| 命令 | 预占、释放、确认出库、确认入库、冻结、解冻、调整 |
| 领域事件 | 预占成功、释放成功、扣减完成、入库完成、冻结完成、可售变化 |
| 查询模型 | 库存余额、可售库存、预占记录、库存流水、单据追踪 |
| 不变量 | 不超卖、不超释放、不超扣、不改历史流水、余额与流水一致 |

核心原则：

```text
外部系统发起库存命令
  -> 中央库存校验幂等和聚合不变量
  -> InventoryAccount / InventoryReservation 状态变化
  -> 追加 InventoryLedger
  -> 发布 InventoryEvent
  -> 外部系统根据事件更新本地进度快照
```

## 2. 接口分层

| 类型 | 方向 | DDD 语义 | 典型用途 |
| --- | --- | --- | --- |
| 查询接口 | 外部系统 -> 中央库存 | 读模型查询 | OMS 分仓、WMS 排查、采购/调拨查询库存 |
| 命令接口 | 外部系统 -> 中央库存 | 调用聚合行为 | 预占、释放、扣减、入库、冻结、调整 |
| 事件接口 | 中央库存 -> 外部系统 | 发布领域事实 | 通知库存动作已经成功发生 |
| 对账接口 | 双向 | 读模型与事实核对 | 按单据、仓库、SKU、流水批次排查差异 |

## 3. 通用命令请求

所有改变库存的接口都必须携带这些字段。

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `request_id` | 是 | 链路追踪 ID |
| `idempotency_key` | 是 | 幂等键，同一业务动作唯一 |
| `source_system` | 是 | 来源系统：OMS、WMS、PURCHASE、TRANSFER、AFTERSALE、SRM |
| `source_doc_type` | 是 | 来源单据类型 |
| `source_doc_no` | 是 | 来源单号 |
| `source_doc_line_no` | 否 | 来源行号 |
| `operator_id` | 否 | 操作人或系统账号 |
| `occurred_at` | 是 | 业务发生时间 |
| `reason_code` | 否 | 业务原因 |
| `remark` | 否 | 备注 |

命令响应建议统一：

| 字段 | 说明 |
| --- | --- |
| `success` | 是否成功 |
| `code` | 业务错误码 |
| `message` | 返回说明 |
| `request_id` | 原请求 ID |
| `event_id` | 本次库存事件 ID |
| `ledger_batch_no` | 本次流水批次 |
| `idempotent_hit` | 是否命中幂等 |
| `results` | 行级处理结果 |

行级结果包含 `owner_id`、`warehouse_id`、`sku_id`、`lot_id`、`requested_qty`、`processed_qty`、`before_balance`、`after_balance`、`reservation_id`、`line_result`。

## 4. 查询接口

查询接口只读取中央库存读模型，不改变聚合状态。

### 4.1 查询库存余额

```text
GET /inventory/balances
```

| 参数 | 说明 |
| --- | --- |
| `owner_id` | 货主/公司 |
| `warehouse_ids` | 仓库范围 |
| `sku_ids` | SKU 范围 |
| `lot_ids` | 批次范围，可选 |
| `inventory_status` | 库存状态 |
| `include_zero` | 是否返回零库存 |

返回 `on_hand_qty`、`available_qty`、`reserved_qty`、`frozen_qty`、`in_transit_qty`、`non_sellable_qty`、`version_no`、`updated_at`。

### 4.2 查询可售库存

```text
POST /inventory/sellable/query
```

可售库存是查询策略结果，不一定等于可用库存。它可能扣除安全库存、渠道保留、限售策略和预占。

| 参数 | 说明 |
| --- | --- |
| `channel_id` | 销售渠道 |
| `owner_id` | 货主 |
| `warehouse_scope` | 可售仓范围 |
| `sku_ids` | SKU 列表 |
| `strategy_code` | 可售策略 |

### 4.3 查询预占记录

```text
GET /inventory/reservations
```

返回 `reservation_id`、`reservation_type`、`source_doc_no`、`reserved_qty`、`released_qty`、`deducted_qty`、`reservation_status`、`expires_at`。

### 4.4 查询库存流水

```text
GET /inventory/ledgers
```

用于排查、审计和对账。常用过滤条件包括 `ledger_type`、`source_doc_type`、`source_doc_no`、`warehouse_id`、`sku_id`、`lot_id`、`ledger_batch_no`、`occurred_from`、`occurred_to`。

### 4.5 查询单据库存轨迹

```text
GET /inventory/traces/by-source-doc
```

按来源单据串起预占、释放、扣减、入库、调整、事件和流水批次，适合客服、运营和技术排障。

## 5. 命令接口

### 5.1 预占库存 `reserveStock`

```text
POST /inventory/reservations
```

| 字段 | 说明 |
| --- | --- |
| `reservation_type` | `SALE`、`TRANSFER`、`SUPPLIER_RETURN` |
| `owner_id` | 货主 |
| `warehouse_id` | 预占仓 |
| `expires_at` | 预占过期时间 |
| `lines[].sku_id` | SKU |
| `lines[].lot_id` | 批次，可选 |
| `lines[].qty` | 预占数量 |
| `lines[].inventory_status` | 可用、不可售、冻结等 |

不变量：

| 规则 | 说明 |
| --- | --- |
| 不超占 | 销售/调拨默认要求 `available_qty >= qty` |
| 退供可配置库存状态 | 退供可锁定可用、不可售或冻结库存 |
| 预占必须可追溯来源 | 每条预占必须绑定来源单据和行 |
| 幂等内容一致 | 同一幂等键重复调用，参数必须一致 |

成功后发布 `InventoryReserved`，并生成 `SALE_RESERVE`、`TRANSFER_RESERVE` 或 `SUPPLIER_RETURN_RESERVE` 流水。

### 5.2 释放预占 `releaseReservation`

```text
POST /inventory/reservations/{reservation_id}/release
```

| 字段 | 说明 |
| --- | --- |
| `release_reason` | `CANCEL`、`SHORTAGE`、`TIMEOUT`、`SUPPLIER_REJECT`、`MANUAL` |
| `lines[].sku_id` | SKU |
| `lines[].qty` | 释放数量 |

不变量：累计释放数量不能超过剩余预占数量；已扣减部分不能释放；重复释放不能重复增加可用库存。成功后发布 `InventoryReservationReleased`。

### 5.3 确认出库 `confirmOutbound`

```text
POST /inventory/outbounds/confirm
```

适用于销售出库、调拨出库、退供应商出库。

| 字段 | 说明 |
| --- | --- |
| `outbound_type` | `SALE`、`TRANSFER`、`SUPPLIER_RETURN`、`OTHER` |
| `reservation_id` | 关联预占，销售/调拨/退供建议必填 |
| `warehouse_id` | 出库仓 |
| `package_no` | 包裹号，可选 |
| `tracking_no` | 运单号，可选 |
| `lines[].sku_id` | SKU |
| `lines[].lot_id` | 批次，可选 |
| `lines[].shipped_qty` | 实发数量 |

| 出库类型 | 库存影响 |
| --- | --- |
| `SALE` | 减少出库仓 `on_hand_qty`，冲减销售预占 |
| `TRANSFER` | 减少调出仓 `on_hand_qty`，冲减调拨预占，增加 `in_transit_qty` |
| `SUPPLIER_RETURN` | 减少企业库存，冲减退供锁定 |
| `OTHER` | 按配置生成其他出库或调整流水 |

不变量：实发不能超过可扣数量；重复发货回传幂等；实发小于预占时必须明确剩余预占释放、保留或异常关闭策略。成功后发布 `InventoryDeducted`。

### 5.4 确认入库 `confirmInbound`

```text
POST /inventory/inbounds/confirm
```

适用于采购入库、调拨入库、售后退货入库。

| 字段 | 说明 |
| --- | --- |
| `inbound_type` | `PURCHASE`、`TRANSFER`、`RETURN_SELLABLE`、`RETURN_NON_SELLABLE`、`OTHER` |
| `warehouse_id` | 入库仓 |
| `source_outbound_no` | 调拨入库可关联调拨出库 |
| `lines[].sku_id` | SKU |
| `lines[].lot_id` | 批次，可选 |
| `lines[].putaway_qty` | 上架数量 |
| `lines[].inventory_status` | 可用、不可售、冻结 |

| 入库类型 | 库存影响 |
| --- | --- |
| `PURCHASE` | 增加入库仓实物和可用库存 |
| `TRANSFER` | 减少在途，增加调入仓实物和可用库存 |
| `RETURN_SELLABLE` | 增加退货仓实物和可用库存 |
| `RETURN_NON_SELLABLE` | 增加退货仓实物和不可售/冻结库存 |
| `OTHER` | 按配置生成其他入库或调整流水 |

成功后发布 `InventoryInboundConfirmed`。

### 5.5 冻结/解冻库存

```text
POST /inventory/freeze
POST /inventory/unfreeze
```

冻结用于质检、盘点、风控、人工冻结；解冻用于质检通过、盘点结束或风控解除。不变量：冻结数量不能超过可冻结数量，解冻数量不能超过已冻结数量。成功后发布 `InventoryFrozen` 或 `InventoryUnfrozen`。

### 5.6 库存调整

```text
POST /inventory/adjustments
```

调整用于盘盈、盘亏、报废、异常修正。调整命令风险高，必须绑定审批单据。

| 字段 | 说明 |
| --- | --- |
| `adjust_type` | `ADJUST_IN`、`ADJUST_OUT`、`SCRAP`、`LOSS`、`CORRECTION` |
| `approval_doc_no` | 审批单号 |
| `reason_code` | 调整原因 |
| `lines[].qty` | 调整数量 |

规则：错误流水不改历史，只能追加反向或修正流水。成功后发布 `InventoryAdjusted`。

## 6. 领域事件契约

库存命令成功后发布领域事件。事件是已经发生的事实，消费者不能把事件当成命令再次要求库存系统修改库存。

| 事件 | 触发命令 | 关键消费者 |
| --- | --- | --- |
| `InventoryReserved` | `reserveStock` | OMS、调拨、退供 |
| `InventoryReservationReleased` | `releaseReservation` | OMS、调拨、退供 |
| `InventoryDeducted` | `confirmOutbound` | OMS、调拨、采购/SRM、BMS |
| `InventoryInboundConfirmed` | `confirmInbound` | 采购、调拨、售后、BMS |
| `InventoryFrozen` | `freezeStock` | WMS、质检、报表 |
| `InventoryUnfrozen` | `unfreezeStock` | WMS、质检、报表 |
| `InventoryAdjusted` | `adjustStock` | WMS、财务、报表 |
| `SellableInventoryChanged` | 任意影响可售的命令 | OMS、渠道、报表 |

事件通用字段：

| 字段 | 说明 |
| --- | --- |
| `event_id` | 事件 ID |
| `event_type` | 事件类型 |
| `aggregate_type` | `InventoryAccount` 或 `InventoryReservation` |
| `aggregate_id` | 聚合 ID |
| `source_doc_type` / `source_doc_no` | 来源单据 |
| `ledger_batch_no` | 库存流水批次 |
| `owner_id` / `warehouse_id` / `sku_id` / `lot_id` | 库存维度 |
| `changed_fields` | 变化字段 |
| `occurred_at` | 业务发生时间 |
| `version_no` | 聚合版本 |

## 7. 错误码

| 错误码 | 场景 |
| --- | --- |
| `INV_INSUFFICIENT_AVAILABLE` | 可用库存不足 |
| `INV_RESERVATION_NOT_FOUND` | 预占不存在 |
| `INV_RESERVATION_CLOSED` | 预占已释放或已冲减 |
| `INV_OVER_RELEASE` | 释放数量超过剩余预占 |
| `INV_OVER_DEDUCT` | 扣减数量超过预占或可扣数量 |
| `INV_BALANCE_NOT_FOUND` | 库存账户不存在 |
| `INV_LOT_NOT_ALLOWED` | 批次不允许使用 |
| `INV_STATUS_NOT_ALLOWED` | 库存状态不可操作 |
| `INV_IDEMPOTENCY_CONFLICT` | 同幂等键但请求内容不一致 |
| `INV_CONCURRENT_CONFLICT` | 并发更新冲突 |
| `INV_ADJUSTMENT_NOT_APPROVED` | 调整单未审批 |

## 8. 一致性与补偿

| 规则 | 说明 |
| --- | --- |
| 命令必须幂等 | 所有改变库存的接口必须带 `idempotency_key` |
| 余额与流水同事务 | 更新余额、追加流水、记录事件必须原子提交 |
| 同一库存账户串行更新 | 同一货主、仓、SKU、批次、状态需要版本控制或锁 |
| 事件后置发布 | 事务提交成功后才能发布事件 |
| 外部回传可重放 | WMS 发货、上架、验收回传失败后可以重复发送 |
| 异常不能静默修正 | 多发、多收、少收、丢失必须走异常、调整或人工审批 |

## 9. 典型业务映射

| 业务流程 | 库存命令 | 主要事件 |
| --- | --- | --- |
| 销售订单审核通过 | `reserveStock(SALE)` | `InventoryReserved` |
| 销售订单取消 | `releaseReservation(CANCEL)` | `InventoryReservationReleased` |
| WMS 销售发货 | `confirmOutbound(SALE)` | `InventoryDeducted` |
| 采购上架 | `confirmInbound(PURCHASE)` | `InventoryInboundConfirmed` |
| 调拨审核通过 | `reserveStock(TRANSFER)` | `InventoryReserved` |
| 调出仓发货 | `confirmOutbound(TRANSFER)` | `InventoryDeducted` |
| 调入仓上架 | `confirmInbound(TRANSFER)` | `InventoryInboundConfirmed` |
| 售后可售退货上架 | `confirmInbound(RETURN_SELLABLE)` | `InventoryInboundConfirmed` |
| 售后不可售退货入库 | `confirmInbound(RETURN_NON_SELLABLE)` | `InventoryInboundConfirmed` |
| 退供审核通过 | `reserveStock(SUPPLIER_RETURN)` | `InventoryReserved` |
| 退供出库 | `confirmOutbound(SUPPLIER_RETURN)` | `InventoryDeducted` |

## 11. 第一版优先级

| 优先级 | 接口 | 原因 |
| --- | --- | --- |
| P0 | 查询库存余额 | 所有流程依赖库存查询 |
| P0 | 预占库存 | 销售履约防超卖 |
| P0 | 释放预占 | 取消、缺货、超时必须释放 |
| P0 | 确认出库 | 发货扣减库存 |
| P0 | 确认入库 | 采购、退货、调拨入库增加库存 |
| P1 | 查询库存流水 | 排查和对账 |
| P1 | 冻结/解冻 | 质检、盘点、风控 |
| P1 | 库存调整 | 盘点、报废、差异修正 |
| P2 | 对账接口 | 系统稳定后加强治理 |
| P2 | 可售库存策略接口 | 渠道库存精细化后扩展 |

## 12. 继续上下文

当前结论：库存接口分为查询、命令、事件、对账四类；命令驱动 `InventoryAccount` 和 `InventoryReservation` 聚合，成功后追加流水并发布事件。

关键假设：中央库存是库存余额、库存预占、库存流水的权威系统；WMS 是仓内实物作业事实来源；OMS、采购、调拨、售后、退供系统只保存库存处理结果快照。

下一步：若进入技术落地，建议继续输出库存 OpenAPI、库存表结构 DDL，以及库存并发和补偿专项设计。
