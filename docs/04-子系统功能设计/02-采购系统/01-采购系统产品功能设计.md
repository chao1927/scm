# 01-采购系统产品功能设计

> 本文是 `docs/04-子系统功能设计/采购系统` 的唯一产品功能设计文档，从产品视角描述菜单、页面、按钮、操作逻辑、角色权限、数据权限、查询、列表、详情、新增、修改、登录/登出和审计。领域规则参考 `docs/03-核心业务模型/02-采购领域模型`，表字段参考 `docs/05-子系统数据库设计/02-采购系统数据库设计.md`。

## 1. 产品定位与设计边界

| 项 | 设计 |
| --- | --- |
| 系统 | 采购系统 |
| 设计对象 | 后台管理页面、业务操作页面、配置页面、日志页面和权限控制 |
| 不承载内容 | 数据库 DDL、领域聚合细节、跨系统接口实现；这些分别在数据库设计、核心业务模型、接口事件文档维护 |
| 产品原则 | 先通过权限系统完成登录和授权，再按菜单进入功能；所有写操作必须校验按钮权限、数据权限、状态机和幂等键 |

## 2. 登录、登出与全局交互

| 页面/动作 | 使用角色 | 展示字段 | 操作逻辑 |
| --- | --- | --- | --- |
| 登录页 | 未登录用户 | 账号、密码、验证码/MFA、租户/组织可选 | 用户输入凭证后调用权限系统认证；成功写入 access_token/refresh_token，拉取菜单、按钮权限和数据范围；失败记录登录日志。 |
| 退出登录 | 已登录用户 | 当前用户、当前系统、会话过期时间 | 用户点击退出后吊销当前 Token，清理前端缓存、菜单缓存和权限缓存，并跳转登录页。 |
| 个人中心 | 已登录用户 | 姓名、账号、手机号、邮箱、角色、所属组织 | 读取当前用户资料和角色权限；修改密码或个人资料时记录操作日志。 |

全局页面规则：

- 所有列表页默认包含查询、重置、分页、排序、导出入口；导出必须受数据权限约束。
- 所有详情页默认包含基础信息、明细信息、状态流转、领域事件、操作日志。
- 所有新增/编辑页默认校验必填、唯一、枚举、引用关系、乐观锁和状态机。
- 所有写操作默认记录操作日志；会改变业务事实的操作写入领域事件发布表。

## 3. 角色、菜单权限与数据权限

| 角色 | 可用功能 | 典型动作 | 数据范围 |
| --- | --- | --- | --- |
| 需求部门 | 请购 | 提交采购申请、查看进度 | 本人、本部门或被分配业务范围 |
| 采购员 | 询价、下单、跟单 | 创建询价、比价、生成采购订单 | 本人、本部门或被分配业务范围 |
| 采购经理 | 审批、供应商选择 | 审核采购申请和采购订单 | 本组织及下级组织，或审批授权范围 |
| 质量人员 | 质量要求 | 维护质检要求、查看不良 | 本人、本部门或被分配业务范围 |
| 财务人员 | 价格/税率/对账查看 | 查看采购金额、应付依据 | 按账套、组织、客户/供应商、货主授权 |
| 系统管理员 | 参数配置 | 采购类型、审批规则、原因码 | 按授权组织和系统配置范围 |

数据权限维度：

| 维度 | 控制方式 | 页面效果 |
| --- | --- | --- |
| 组织 | 按当前组织、下级组织或自定义组织集合过滤 | 列表、详情、导出只返回可见组织数据 |
| 仓库 | WMS、库存、OMS、BMS 按仓库授权过滤 | 未授权仓库不可查询、不可操作 |
| 货主 | 多货主场景按货主授权过滤 | 只能查看和操作授权货主的单据和库存 |
| 供应商/客户 | 外部协同用户绑定供应商或客户 | 只展示自身协同单据，不展示其他主体数据 |
| 本人/部门 | 普通业务用户可限制为本人或本部门 | 请购、待办、审批等页面优先使用该范围 |

## 7. 关键操作逻辑

| 操作类型 | 前置校验 | 状态/数据变化 | 事件与审计 |
| --- | --- | --- | --- |
| 新增 | 菜单权限、按钮权限、必填、唯一、引用主数据有效 | 创建草稿或初始状态数据 | 记录操作日志；必要时发布已创建事件 |
| 修改 | 按钮权限、数据权限、乐观锁、非终态、字段可编辑 | 更新业务字段或生成变更记录 | 记录前后快照；关键变更发布事件 |
| 删除 | 通常只允许草稿或未引用数据 | 逻辑删除，历史单据不物理删除 | 记录删除原因和操作者 |
| 提交/发布/审批 | 状态机允许、审批权限、幂等键未重复 | 推进状态，锁定或发布业务事实 | 写领域事件表，刷新读模型 |
| 取消/关闭/停用 | 无未完成依赖或允许补偿路径 | 进入终态或禁用状态 | 记录原因；通知下游或生成待办 |
| 查询/导出 | 查询权限、数据范围、脱敏策略 | 不改变业务状态 | 导出记录审计日志 |

## 10. TMS 与业务流程协同

采购系统不拥有运输事实，但采购到货、供应商发运和退供应商出库需要展示 TMS 的运输状态，并在必要时向 TMS 发起运输请求。

| 页面 | TMS协同字段/功能 | 操作按钮 | 权限点 |
| --- | --- | --- | --- |
| 采购订单列表/详情 | 运单号、承运商、预计到仓时间、最新轨迹、运输异常标记 | 查看轨迹、查看异常 | `purchase:tms_track:read` |
| ASN/到货跟踪 | 供应商发运时间、承运商单号、在途状态、到仓状态 | 同步运输信息、查看轨迹 | `purchase:asn_transport:read` |
| 退供应商单 | 退供运单号、发货仓库、收货供应商、退供签收结果 | 请求退供运输、查看退供轨迹 | `purchase:return_transport:request` |
| 采购异常处理 | 延误、破损、丢失、拒收等物流异常 | 关联物流异常、提交索赔说明 | `purchase:tms_exception:handle` |

| 事件 | 来源/去向 | 采购系统处理 |
| --- | --- | --- |
| ASN已提交 | 采购/供应商系统 -> TMS | TMS 可据此生成采购到货运输跟踪；采购系统保留 ASN 与运单关联 |
| 运输已到达 | TMS -> 采购系统 | 更新采购到货读模型，提示仓库准备收货，不直接改变采购订单完成状态 |
| 运输已签收 | TMS -> 采购系统 | 更新退供签收或采购到货运输完成状态 |
| 运输已拒收 | TMS -> 采购系统 | 生成采购物流异常待办，退供场景进入异常处理 |
| 物流异常已登记 | TMS -> 采购系统 | 更新采购异常看板和采购订单详情时间线 |

TMS 与业务流程的覆盖口径如下，已纳入本系统页面、按钮、事件和权限设计：


| 来源流程    | 已纳入的产品能力                  | 页面/操作落点                         | 状态或事件影响                    |
| ------- | -------------------------- | ------------------------------- | -------------------------- |
| 采购入库    | 采购订单、ASN、到货计划要能关联运输任务和预计到仓 | 采购订单详情、ASN/到货跟踪增加运输状态、预计到仓、到达提醒 | 消费`运输已到达`只更新到货准备，不关闭采购订单   |
| 供应商退货   | 退供应商单要能发起退供运输、查看供应商签收/拒收   | 退供应商单增加请求退供运输、查看轨迹、登记拒收处理       | 消费`运输已签收/拒收`推进退供进度或生成异常待办  |
| TMS运输协同 | 采购物流异常要能回到采购异常闭环           | 采购异常处理增加关联运单、异常责任方、索赔说明         | 消费`物流异常已登记/已关闭`刷新采购异常和索赔依据 |
| BMS费用结算 | 采购运费、退供运费和索赔要可追溯           | 采购订单/退供单详情展示费用来源号、BMS费用状态       | 消费费用状态读模型，不直接生成最终账单        |

当前结论：本文是产品功能设计入口；后续页面原型、接口定义和后端实现都应以本文的菜单、按钮、权限和操作逻辑为准。
