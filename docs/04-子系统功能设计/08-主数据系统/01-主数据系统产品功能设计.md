# 01-主数据系统产品功能设计

> 本文是 `docs/04-子系统功能设计/08-主数据系统` 的唯一产品功能设计文档，从产品视角描述菜单、页面、按钮、操作逻辑、角色权限、数据权限、查询、列表、详情、新增、修改、登录/登出和审计。领域规则参考 `docs/03-核心业务模型/08-主数据领域模型`，表字段参考 `docs/05-子系统数据库设计/08-主数据系统数据库设计.md`。

## 1. 产品定位与设计边界

| 项 | 设计 |
| --- | --- |
| 系统 | 主数据系统 |
| 设计对象 | 后台管理页面、业务操作页面、配置页面、日志页面和权限控制 |
| 不承载内容 | 数据库 DDL、领域聚合细节、跨系统接口实现；这些分别在数据库设计、核心业务模型、接口事件文档维护 |
| 产品原则 | 先通过权限系统完成登录和授权，再按菜单进入功能；所有写操作必须校验按钮权限、数据权限、状态机和幂等键 |

## 2. 登录、登出与全局交互

| 页面/动作 | 使用角色 | 展示字段 | 操作逻辑 |
| --- | --- | --- | --- |
| 登录页 | 未登录用户 | 账号、密码、验证码/MFA、租户/组织可选 | 用户输入凭证后调用权限系统认证；成功写入 access_token/refresh_token，拉取菜单、按钮权限和数据范围；失败记录登录日志。 |
| 退出登录 | 已登录用户 | 当前用户、当前系统、会话过期时间 | 用户点击退出后吊销当前 Token，清理前端缓存、菜单缓存和权限缓存，并跳转登录页。 |
| 个人中心 | 已登录用户 | 姓名、账号、手机号、邮箱、角色、所属组织 | 读取当前用户资料和角色权限；修改密码或个人资料时记录操作日志。 |

全局页面规则：

- 所有列表页默认包含查询、重置、分页、排序、导出入口；导出必须受数据权限约束。
- 所有详情页默认包含基础信息、明细信息、状态流转、领域事件、操作日志。
- 所有新增/编辑页默认校验必填、唯一、枚举、引用关系、乐观锁和状态机。
- 所有写操作默认记录操作日志；会改变业务事实的操作写入领域事件发布表。

## 3. 角色、菜单权限与数据权限

| 角色 | 可用功能 | 典型动作 | 数据范围 |
| --- | --- | --- | --- |
| 主数据专员 | 建档、维护、审核流转 | 创建 SKU、供应商、仓库、物流商 | 本人、本部门或被分配业务范围 |
| 商品运营 | 商品资料 | 维护 SPU/SKU、属性、图片 | 本人、本部门或被分配业务范围 |
| 采购/供应商管理 | 供应商资料 | 供应商准入、供应商商品 | 仅本人绑定供应商数据 |
| 仓储运营 | 仓库库位 | 新仓、库区、库位、货主仓关系 | 按仓库、货主、库区授权 |
| 物流运营 | 物流商 | 物流产品、服务区域、面单 | 本人、本部门或被分配业务范围 |
| 财务 | 税率、结算资料 | 审核税号、账期、账户 | 按账套、组织、客户/供应商、货主授权 |
| 系统管理员 | 编码、模板、权限 | 配置编码规则、审批规则 | 按授权组织和系统配置范围 |

数据权限维度：

| 维度 | 控制方式 | 页面效果 |
| --- | --- | --- |
| 组织 | 按当前组织、下级组织或自定义组织集合过滤 | 列表、详情、导出只返回可见组织数据 |
| 仓库 | WMS、库存、OMS、BMS 按仓库授权过滤 | 未授权仓库不可查询、不可操作 |
| 货主 | 多货主场景按货主授权过滤 | 只能查看和操作授权货主的单据和库存 |
| 供应商/客户 | 外部协同用户绑定供应商或客户 | 只展示自身协同单据，不展示其他主体数据 |
| 本人/部门 | 普通业务用户可限制为本人或本部门 | 请购、待办、审批等页面优先使用该范围 |

| 菜单路由 | `/mdm/types` |
| 主要用户 | 业务操作员、业务主管、系统管理员；外部协同页面按供应商/客户/货主数据范围过滤 |

| 页面目的 | 配置有哪些主数据 |

| 区域 | 字段/按钮 | 规则 |
| --- | --- | --- |
| 查询条件 | 类型编码、类型名称、状态、主数据类型ID、上级类型、启用、停用、排序 | 支持组合查询；未选条件默认按用户数据权限过滤 |
| 排序 | 状态 | 默认按更新时间倒序；支持分页，每页 10/20/50 条 |
| 列表展示 | 类型编码、类型名称、所属领域、状态、主数据类型ID、商品、伙伴、仓储、物流、组织、财务、上级类型、是否需要审核、是否启用版本、是否发布给子系统 | 列表展示业务识别字段、状态、关键金额/数量和更新时间 |
| 顶部按钮 | 查询、重置、新增 | 根据按钮权限和数据范围显示 |
| 行内操作 | 查看详情、编辑、启用、停用 | 终态数据只允许查看、导出或复制，不允许直接编辑 |
| 详情展示 | 主数据类型ID、类型编码、类型名称、商品、伙伴、仓储、物流、组织、财务、上级类型、是否需要审核、是否启用版本、是否发布给子系统、启用、停用、排序、说明 | 详情页展示基础信息、明细、状态流转、事件记录和操作日志 |
| 新增页面 | 主数据类型ID、类型编码、类型名称、商品、伙伴、仓储、物流、组织、财务、上级类型、是否需要审核、是否启用版本、是否发布给子系统、排序、说明 | 新增时校验必填、唯一、枚举、引用关系和数据权限 |
| 修改页面 | 主数据类型ID、类型编码、类型名称、商品、伙伴、仓储、物流、组织、财务、上级类型、是否需要审核、是否启用版本、是否发布给子系统、排序、说明 | 已生效/终态数据只允许走变更、关闭、停用或补偿流程 |

| 操作 | 权限点 | 操作逻辑 | 结果/审计 |
| --- | --- | --- | --- |
| 查询 | `master-data:master_data:read` | 读取读模型或列表投影，不修改业务状态 | 不记录业务事件，可记录访问日志 |
| 新增 | `master-data:master_data:create` | 打开新增页，提交后由后端应用服务校验并创建聚合 | 记录操作日志，必要时发布已创建事件 |
| 编辑 | `master-data:master_data:update` | 加载当前版本，校验乐观锁、状态机和数据权限后保存 | 记录前后快照，必要时发布已变更事件 |
| 启用 | `master-data:master_data:enable` | 执行状态推进命令；状态流转必须符合领域模型状态机 | 写操作日志，写领域事件表，刷新列表读模型 |
| 停用 | `master-data:master_data:disable` | 执行状态推进命令；状态流转必须符合领域模型状态机 | 写操作日志，写领域事件表，刷新列表读模型 |

## 7. 关键操作逻辑

## 10. TMS 与业务流程协同

主数据系统为 TMS 提供物流商、物流产品、地址区域、服务范围、禁运规则和面单模板等基础资料。TMS 消费主数据事件后形成本系统可用的承运商和规则读模型。

| 页面 | TMS协同字段/功能 | 操作按钮 | 权限点 |

| 调拨 | 调出仓、调入仓、仓间线路和物流产品要被主数据管理 | 仓库、物流产品、服务范围页增加仓间线路适配 | 支撑 TMS 创建调拨运输任务和 BMS 计费规则 |
| BMS费用结算 | 物流商、计费对象、税率、币种要提供权威口径 | 物流商、计费对象、税率币种页增加结算主体映射 | 发布主数据变更后 BMS/TMS 刷新费用口径 |

当前结论：本文是产品功能设计入口；后续页面原型、接口定义和后端实现都应以本文的菜单、按钮、权限和操作逻辑为准。
