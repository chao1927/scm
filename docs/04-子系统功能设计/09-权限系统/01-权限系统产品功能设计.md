# 01-权限系统产品功能设计

> 本文是 `docs/04-子系统功能设计/09-权限系统` 的唯一产品功能设计文档，从产品视角描述菜单、页面、按钮、操作逻辑、角色权限、数据权限、查询、列表、详情、新增、修改、登录/登出和审计。领域规则参考 `docs/03-核心业务模型/09-权限系统领域模型`，表字段参考 `docs/05-子系统数据库设计/09-权限系统数据库设计.md`。

## 1. 产品定位与设计边界

| 项 | 设计 |
| --- | --- |
| 系统 | 权限系统 |
| 设计对象 | 后台管理页面、业务操作页面、配置页面、日志页面和权限控制 |
| 不承载内容 | 数据库 DDL、领域聚合细节、跨系统接口实现；这些分别在数据库设计、核心业务模型、接口事件文档维护 |
| 产品原则 | 先通过权限系统完成登录和授权，再按菜单进入功能；所有写操作必须校验按钮权限、数据权限、状态机和幂等键 |

## 2. 登录、登出与全局交互

| 页面/动作 | 使用角色 | 展示字段 | 操作逻辑 |
| --- | --- | --- | --- |
| 登录页 | 未登录用户 | 账号、密码、验证码/MFA、租户/组织可选 | 用户输入凭证后调用权限系统认证；成功写入 access_token/refresh_token，拉取菜单、按钮权限和数据范围；失败记录登录日志。 |
| 退出登录 | 已登录用户 | 当前用户、当前系统、会话过期时间 | 用户点击退出后吊销当前 Token，清理前端缓存、菜单缓存和权限缓存，并跳转登录页。 |
| 个人中心 | 已登录用户 | 姓名、账号、手机号、邮箱、角色、所属组织 | 读取当前用户资料和角色权限；修改密码或个人资料时记录操作日志。 |

全局页面规则：

- 所有列表页默认包含查询、重置、分页、排序、导出入口；导出必须受数据权限约束。
- 所有详情页默认包含基础信息、明细信息、状态流转、领域事件、操作日志。
- 所有新增/编辑页默认校验必填、唯一、枚举、引用关系、乐观锁和状态机。
- 所有写操作默认记录操作日志；会改变业务事实的操作写入领域事件发布表。

## 3. 角色、菜单权限与数据权限

| 角色 | 可用功能 | 典型动作 | 数据范围 |
| --- | --- | --- | --- |
| 系统管理员 | 用户、角色、权限 | 创建用户、分配角色 | 按授权组织和系统配置范围 |
| 组织管理员 | 部门、岗位、人员 | 维护组织和岗位 | 按授权组织和系统配置范围 |
| 业务主管 | 审批配置、授权 | 配置采购/库存/主数据审批 | 本组织及下级组织，或审批授权范围 |
| 审批人 | 待办审批 | 审批订单、调整、主数据变更 | 本组织及下级组织，或审批授权范围 |
| 审计人员 | 日志查询 | 查看敏感操作和权限变更 | 本人、本部门或被分配业务范围 |
| 普通用户 | 登录、待办 | 使用被授权功能 | 本人、本部门或被分配业务范围 |

数据权限维度：

| 维度 | 控制方式 | 页面效果 |
| --- | --- | --- |
| 组织 | 按当前组织、下级组织或自定义组织集合过滤 | 列表、详情、导出只返回可见组织数据 |
| 仓库 | WMS、库存、OMS、BMS 按仓库授权过滤 | 未授权仓库不可查询、不可操作 |
| 货主 | 多货主场景按货主授权过滤 | 只能查看和操作授权货主的单据和库存 |
| 供应商/客户 | 外部协同用户绑定供应商或客户 | 只展示自身协同单据，不展示其他主体数据 |
| 本人/部门 | 普通业务用户可限制为本人或本部门 | 请购、待办、审批等页面优先使用该范围 |

## 4. 左侧菜单与页面总览

| 菜单 | 路由 | 页面目的 | 主要按钮 |
| --- | --- | --- | --- |
| 登录页 | `/iam/login` | SSO 登录入口 | 登录、忘记密码 |
| 用户管理 | `/iam/user` | 用户 CRUD、分配角色 | 新增、编辑、启用、停用、重置密码、分配角色 |
| 角色管理 | `/iam/role` | 角色 CRUD、分配权限 | 新增、编辑、启用、停用、分配权限、查看用户 |
| 菜单页面管理 | `/iam/menu` | 系统菜单和页面维护 | 新增菜单、编辑、调整排序、停用 |
| 权限点管理 | `/iam/permission` | 功能/API/按钮权限维护 | 新增、编辑、绑定菜单、绑定 API |
| 角色授权页 | `/iam/role` | 给角色分配权限 | 勾选授权、取消授权、保存 |
| 用户角色页 | `/iam/userrole` | 给用户分配角色 | 分配角色、取消角色 |
| 会话管理 | `/iam/session` | Token 和在线用户 | 强制下线、查看详情 |
| 操作日志 | `/iam/operation-log` | 查询写操作和敏感操作 | 查询、导出 |
| 枚举配置 | `/iam/enumsettings` | 维护页面枚举项 | 新增、编辑、排序、停用 |

## 5. 页面详细设计

### 登录页

| 项 | 设计 |
| --- | --- |
| 菜单路由 | `/iam/login` |
| 主要用户 | 业务操作员、业务主管、系统管理员；外部协同页面按供应商/客户/货主数据范围过滤 |
| 业务对象 | 登录日志 |
| 关联表/读模型 | `iam_login_log` |
| 页面目的 | SSO 登录入口 |
| 菜单权限 | `iam:login:read` |

| 区域 | 字段/按钮 | 规则 |
| --- | --- | --- |
| 查询条件 | 登录账号 | 支持组合查询；未选条件默认按用户数据权限过滤 |
| 排序 | 创建时间、更新时间 | 默认按更新时间倒序；支持分页，每页 10/20/50 条 |
| 列表展示 | 账号、密码、验证码、MFA、登录日志ID、登录账号、成功、失败、密码错误、用户停用、Token 过期等、IP、客户端信息 | 列表展示业务识别字段、状态、关键金额/数量和更新时间 |
| 顶部按钮 | 查询、重置 | 根据按钮权限和数据范围显示 |
| 行内操作 | 查看详情、登录、忘记密码 | 终态数据只允许查看、导出或复制，不允许直接编辑 |
| 详情展示 | 登录日志ID、登录账号、成功、失败、密码错误、用户停用、Token 过期等、IP、客户端信息、登录时间 | 详情页展示基础信息、明细、状态流转、事件记录和操作日志 |
| 新增页面 | 登录日志ID、登录账号、成功、失败、密码错误、用户停用、Token 过期等、IP、客户端信息、登录时间 | 新增时校验必填、唯一、枚举、引用关系和数据权限 |
| 修改页面 | 登录日志ID、登录账号、成功、失败、密码错误、用户停用、Token 过期等、IP、客户端信息、登录时间 | 已生效/终态数据只允许走变更、关闭、停用或补偿流程 |

| 操作 | 权限点 | 操作逻辑 | 结果/审计 |
| --- | --- | --- | --- |
| 查询 | `iam:login:read` | 读取读模型或列表投影，不修改业务状态 | 不记录业务事件，可记录访问日志 |
| 登录 | `iam:login:login` | 执行业务动作，后端校验状态、权限、幂等键和参数 | 记录操作日志，按需要发布领域事件 |
| 忘记密码 | `iam:login:page` | 执行业务动作，后端校验状态、权限、幂等键和参数 | 记录操作日志，按需要发布领域事件 |

## 7. 关键操作逻辑

| 操作类型 | 前置校验 | 状态/数据变化 | 事件与审计 |
| --- | --- | --- | --- |
| 新增 | 菜单权限、按钮权限、必填、唯一、引用主数据有效 | 创建草稿或初始状态数据 | 记录操作日志；必要时发布已创建事件 |
| 修改 | 按钮权限、数据权限、乐观锁、非终态、字段可编辑 | 更新业务字段或生成变更记录 | 记录前后快照；关键变更发布事件 |
| 删除 | 通常只允许草稿或未引用数据 | 逻辑删除，历史单据不物理删除 | 记录删除原因和操作者 |
| 提交/发布/审批 | 状态机允许、审批权限、幂等键未重复 | 推进状态，锁定或发布业务事实 | 写领域事件表，刷新读模型 |
| 取消/关闭/停用 | 无未完成依赖或允许补偿路径 | 进入终态或禁用状态 | 记录原因；通知下游或生成待办 |
| 查询/导出 | 查询权限、数据范围、脱敏策略 | 不改变业务状态 | 导出记录审计日志 |

## 10. TMS 与业务流程协同

权限系统需要把 TMS 作为独立应用纳入菜单、按钮、数据权限和审计范围，保证物流专员、仓配人员、客服、采购、财务只能访问各自职责范围内的运输数据。

| 权限对象 | TMS权限点 | 控制内容 |
| --- | --- | --- |
| 菜单权限 | `tms:menu:*` | TMS工作台、运输任务、运单管理、面单管理、轨迹查询、签收管理、物流异常、物流费用来源、配置管理 |
| 按钮权限 | `tms:transport_task:create/update/cancel` | 创建、修改、取消运输任务 |
| 按钮权限 | `tms:waybill:create/cancel/track_sync` | 创建运单、取消运单、同步轨迹 |
| 按钮权限 | `tms:label:generate/print/void` | 生成、打印、作废面单 |
| 按钮权限 | `tms:receipt:correct/proof_upload` | 签收修正、上传签收证明 |
| 按钮权限 | `tms:exception:create/assign/close` | 登记、分派、关闭物流异常 |
| 按钮权限 | `tms:fee_source:generate/push_bms` | 生成物流费用来源、推送 BMS |
| 数据权限 | 组织、仓库、货主、物流商、运输场景、单据归属 | 控制用户可查看和操作的运输数据 |
| 审计权限 | `tms:audit_log:read` | 查询 TMS 操作日志、接口回调日志、费用重推日志 |

| 场景 | 权限系统处理 |
| --- | --- |
| 用户登录 TMS | 返回 TMS 菜单、按钮、数据范围和可访问 API 范围 |
| TMS 写操作 | 校验令牌、按钮权限、数据权限，写入操作日志 |
| 面单补打/签收修正/费用重推 | 作为敏感操作，要求更高按钮权限和原因必填 |
| 跨系统查看轨迹 | OMS、采购、供应商、WMS、BMS 通过各自权限点控制是否可查看 TMS 轨迹 |

TMS 与业务流程的覆盖口径如下，已纳入本系统页面、按钮、事件和权限设计：


| 来源流程 | 已纳入的产品能力 | 页面/操作落点 | 状态或事件影响 |
| --- | --- | --- | --- |
| 全流程 | 登录、菜单、按钮、数据范围要覆盖采购、供应商、WMS、库存、OMS、TMS、BMS、主数据 | 应用、菜单权限、按钮权限、数据权限、用户角色页增加系统维度筛选 | 登录返回各子系统可用功能和数据范围 |
| TMS运输协同 | 物流高危动作需要审批和审计 | 审批配置、操作日志增加换承运商、运单作废、面单作废、轨迹补录、签收冲正、费用修正 | 生产审批结果和审计日志，供 TMS 放行或拒绝 |
| BMS费用结算 | 费用调整、对账差异确认要受权限和审批约束 | 权限点、审批流、操作日志增加费用调整、费用重算、差异确认 | BMS 写操作前校验权限系统 |
| 主数据发布 | 物流商、地址、仓库、货主等数据范围要同步权限对象 | 数据权限页增加物流商、运输场景、仓库、货主、客户/供应商范围 | 消费主数据变更后刷新可授权对象 |

当前结论：本文是产品功能设计入口；后续页面原型、接口定义和后端实现都应以本文的菜单、按钮、权限和操作逻辑为准。
