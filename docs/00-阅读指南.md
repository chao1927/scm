## 3. 新手阅读建议

先读 `01` 和 `02`，只要求知道供应链系统由哪些子系统组成。然后读 `03-供应链系统DDD领域建模与查漏补缺`，用 DDD 视角理解核心域、支撑域、通用域、限界上下文和聚合边界。再读业务流程 `03-1` 到 `07`，把采购入库、销售出库、调拨、退货这几条主线串起来。读完之后再看 `08`、`09`、`16` 和 `50`，因为库存、聚合不变量和领域事件是后续系统详细设计最容易出错、也最需要统一口径的部分。

`03-核心业务模型` 已按领域上下文重组：供应商、采购、WMS、中央库存、OMS、BMS、主数据、权限系统分别有独立领域模型。状态机解决“业务走到哪一步”，字段归属解决“系统要记住哪些信息”，领域模型解决“谁拥有业务语言、聚合和不变量”。

## 4. 最小理解闭环

如果只想快速理解供应链系统的主干，优先读这 6 篇：

```text
01 总览
  -> 02 边界
  -> 03 DDD领域建模
  -> 03-1 采购入库
  -> 03-2 销售出库
  -> 05 库存预占/扣减/释放
  -> 核心聚合与不变量
  -> 中央库存领域模型
  -> 50 上下文映射与领域事件
```

理解这 6 篇后，基本可以看懂后续采购、仓储、库存、调拨、退货和主数据之间的关系。
