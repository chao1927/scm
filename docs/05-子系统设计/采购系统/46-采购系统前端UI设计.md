## 2. 本地静态原型

原型入口：[采购系统原型](../../../prototype/purchase-system/index.html)

原型文件目录：`prototype/purchase-system`

说明：本地原型不依赖后端和 Figma，直接打开 `index.html` 即可查看。每个业务菜单包含列表页、详情页、新增页、修改页，列表支持条件查询、重置、分页和行内操作。

## 3. 已绘制页面

| 页面 | 主要内容 |
| --- | --- |
| 采购工作台 | 待审批请购、待发布询价、待供应商确认、部分入库、退供待审、采购待办和采购闭环 |
| 请购管理 | 请购列表、筛选、状态、模拟数据、增删改查入口 |
| 请购审批 | 审批列表、预算、用途、审批状态、通过/驳回/转交按钮 |
| 询价单 | 询价单列表、供应商数、截止时间、发布/取消/关闭按钮 |
| 报价管理 | 报价单、供应商、报价金额、有效期、录入/导入/确认/作废按钮 |
| 比价定标 | 供应商报价卡、报价、交期、供应商评分、推荐/定标操作 |
| 采购订单 | PO 列表、供应商、金额、订单状态、确认状态、入库进度 |
| 采购订单详情 | 基本信息、商品明细、供应商确认、入库跟踪、操作时间线 |
| 供应商确认处理 | 确认、拒绝、差异反馈和处理状态 |
| 到货跟踪 | ASN、SKU、通知量、收货量、质检/上架状态 |
| 退供申请 | 退供单、供应商、来源 PO、退货仓、原因、审批状态 |
| 采购价格 | 供应商 SKU 价格、税率、币种、生效期、启停状态 |
| 采购参数 | 审批阈值、超收比例、退供原因码等配置项 |
| 操作日志 | 操作人、对象类型、对象单号、动作、结果、来源 IP、操作时间 |
| 枚举配置 | 枚举类型、枚举值、显示名称、排序、启停状态 |

## DDD 对齐说明

本文属于 **采购上下文**。设计时应把页面、字段和流程统一回到该上下文的模型边界，避免跨上下文直接修改数据。

| DDD 项 | 对齐口径 |
| --- | --- |
| 限界上下文 | 采购上下文 |
| 核心聚合 | PurchaseOrder、PurchaseRequisition、SupplierReturnOrder |
| 数据主权 | 采购意图、供应商义务、采购执行进度 |
| 生产事件 | 只发布本上下文已经发生的业务事实 |
| 消费事件 | 消费外部事实时必须记录 event_id、幂等键、处理状态和失败原因 |
| 查询模型 | 列表、看板、导出可使用读模型，不强行加载聚合 |

## 5. 后续上下文

当前结论：采购系统前端页面已经形成第一版本地可运行原型，覆盖采购从请购、询价、报价、比价、采购订单、供应商确认、入库跟踪到退供、价格参数、日志和枚举配置的主要页面。

下一步建议：继续按同一风格绘制 WMS 或 OMS 页面；如果继续细化采购系统，可以优先补充“新增采购订单表单”“请购转采购向导”“供应商差异处理弹窗”“退供申请详情页”。
