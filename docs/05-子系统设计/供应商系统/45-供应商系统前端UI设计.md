## 1. 本地静态原型

原型入口：[供应商系统原型](../../../prototype/supplier-system/index.html)

原型文件目录：`prototype/supplier-system`

说明：本地原型不依赖后端和 Figma，直接打开 `index.html` 即可查看。每个业务菜单包含列表页、详情页、新增页、修改页，列表支持条件查询、重置、分页和行内操作。

## 2. 页面清单

| 页面 | 主要内容 |
| --- | --- |
| 供应商工作台 | 待确认订单、待发货 ASN、退供待确认、对账待确认、评分预警 |
| 供应商档案 | 供应商编码、名称、类型、等级、联系人、结算方式、合作状态 |
| 账号绑定 | 用户、手机号、邮箱、绑定供应商、角色、主账号、启停状态 |
| 供应商商品 | SKU、供应商 SKU、MOQ、MPQ、供货周期、供货状态 |
| 订单协同 | 协同单、PO、供应商、采购员、确认状态、确认截止、差异类型 |
| ASN 管理 | ASN、PO、供应商、目的仓、预计到仓、承运商、发货状态 |
| 退供协同 | 退供协同单、退供单、确认数量、签收数量、差异原因 |
| 对账协同 | 对账单、金额、税额、差异金额、状态、发票号 |
| 质量协同 | 问题单、供应商、SKU、问题类型、严重程度、整改状态 |
| 供应商评分 | 评分周期、综合分、等级、质量分、价格分、交付分、预警原因 |
| 评分规则 | 规则编码、评分维度、权重、公式类型、预警阈值、生效日期 |
| 整改管理 | 整改单、供应商、来源、问题描述、截止时间、审核状态 |
| 操作日志 | 操作人、供应商、对象类型、对象单号、动作、结果、操作时间 |
| 枚举配置 | 枚举类型、枚举值、显示名称、排序、启停状态 |

## DDD 对齐说明

本文属于 **供应商上下文**。设计时应把页面、字段和流程统一回到该上下文的模型边界，避免跨上下文直接修改数据。

| DDD 项 | 对齐口径 |
| --- | --- |
| 限界上下文 | 供应商上下文 |
| 核心聚合 | SupplierProfile、SupplierScore、ASN、SupplierQuote |
| 数据主权 | 供应商准入、协同、评分和供货事实 |
| 生产事件 | 只发布本上下文已经发生的业务事实 |
| 消费事件 | 消费外部事实时必须记录 event_id、幂等键、处理状态和失败原因 |
| 查询模型 | 列表、看板、导出可使用读模型，不强行加载聚合 |
