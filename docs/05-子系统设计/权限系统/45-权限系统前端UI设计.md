## 1. 本地静态原型

原型入口：[权限系统原型](../../../prototype/iam-system/index.html)

说明：本地原型不依赖后端和 Figma，直接打开 `index.html` 即可查看。每个业务菜单包含列表页、详情页、新增页、修改页，列表支持条件查询、重置、分页和行内操作。

## 2. 页面清单

| 页面 | 主要内容 |
| --- | --- |
| 权限工作台 | 在线用户、权限变更、审批待办、登录风险、敏感操作 |
| 应用管理 | 应用编码、应用名称、应用类型、首页地址、启停状态 |
| SSO客户端 | 客户端编码、所属应用、回调地址、授权类型、Token 有效期 |

| 角色管理 | 角色编码、角色名称、角色类型、适用应用、数据范围 |
| 菜单页面 | 应用、父菜单、菜单编码、路由、组件、排序、显示状态 |
| 权限点 | 权限编码、权限名称、权限类型、动作、API 方法、API 路径 |
| 角色授权 | 角色、应用、菜单、权限点、授权状态、授权人 |
| 用户角色 | 用户、角色、授权类型、生效时间、失效时间、授权人 |
| 数据权限 | 授权对象、数据范围、资源类型、资源列表 |
| 审批模板 | 业务类型、审批节点、条件摘要、适用组织、发布状态 |
| 待办中心 | 审批单、业务类型、业务单号、审批人、审批状态 |
| 会话管理 | 用户、客户端、登录 IP、Token 状态、登录时间、过期时间 |
| 登录日志 | 账号、客户端、登录结果、失败原因、IP、登录时间 |
| 操作日志 | 操作人、应用、模块、操作类型、权限点、业务对象、结果 |
| 安全策略 | 密码、锁定、会话、MFA、IP 白名单策略 |
| 枚举配置 | 枚举类型、枚举值、显示名称、默认值、排序、状态 |

## DDD 对齐说明

本文属于 **权限上下文/通用域**。设计时应把页面、字段和流程统一回到该上下文的模型边界，避免跨上下文直接修改数据。

| DDD 项 | 对齐口径 |
| --- | --- |
| 限界上下文 | 权限上下文/通用域 |
| 核心聚合 | User、Role、Permission、Token、OperationLog |
| 数据主权 | 身份、授权、审计和数据权限 |
| 生产事件 | 只发布本上下文已经发生的业务事实 |
| 消费事件 | 消费外部事实时必须记录 event_id、幂等键、处理状态和失败原因 |
| 查询模型 | 列表、看板、导出可使用读模型，不强行加载聚合 |

## 4. 后续上下文

当前结论：权限系统已经形成第一版本地可运行原型，覆盖 SSO 接入、用户、组织、角色、菜单、权限点、角色授权、用户角色、数据权限、审批、Token 会话、登录日志、操作日志、安全策略和枚举配置。
